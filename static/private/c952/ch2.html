2.1 Introduction & Classes of Computing core

Establishes why computer architecture matters and defines the three main categories of computers. Performance depends on algorithm, language/compiler, hardware, and I/O together — no single factor is sufficient.

Personal Computer (PC)Designed for a single user; emphasizes good performance at low cost; runs third-party software. Drove evolution of computing for ~35 years.
ServerRuns larger programs for multiple simultaneous users; accessed via network; emphasizes throughput and dependability. Ranges from low-end (<$1K) to supercomputers.
SupercomputerHighest performance and cost. Tens of thousands of processors, many terabytes of memory, costs tens to hundreds of millions. Used for weather, oil, protein structure.
Embedded ComputerInside another device; runs one predetermined application. Largest class by unit count. Often no visible OS to end user. Low cost/power/failure tolerance.
PMD (Personal Mobile Device)Battery-operated, wirelessly connected, touch/speech input, downloads apps. Smartphones and tablets. Represents the "post-PC era." Passed PCs in units sold in 2011.
Cloud Computing / WSCWarehouse Scale Computers with 100,000+ servers. Companies rent capacity (Amazon, Google). SaaS delivered over the Internet without local install.
SaaSSoftware as a Service — delivered via browser over Internet. No binary installed locally. Examples: web search, social networking.
Multicore MicroprocessorSingle chip containing multiple processor cores. A "quadcore" chip = four processors on one chip.
AcronymWord from initial letters of a string of words. RAM = Random Access Memory. CPU = Central Processing Unit.

2.2 Eight Great Ideas core

Eight foundational principles every computer architect reuses. Matching questions (idea ↔ analogy) appear on most exams. Know all eight names plus a one-line description each.

1. Design for Moore's LawIC resources double every 18–24 months (Gordon Moore, 1965). Design for where technology will be when the project finishes, not when it starts. Analogy: soccer player running to where the ball will be.
2. Use Abstraction to Simplify DesignHide lower-level details behind simpler models at each layer. Fundamental to hardware and software productivity. Analogy: architect designing room layout before door/window details.
3. Make the Common Case FastOptimizing the frequent case yields more gain than optimizing the rare case. Requires measurement to know what is common. Analogy: sports car optimized for 1–2 passengers (most trips).
4. Performance via ParallelismCompute multiple operations simultaneously. Analogy: sister and brother hanging clothes to dry simultaneously.
5. Performance via PipeliningOverlap stages of a task — like a bucket brigade or assembly line. One stage processes while next stage is already working on the previous item. Analogy: one washes dishes, one dries immediately after.
6. Performance via PredictionGuess and start work early; recover cheaply if wrong. Better on average if accuracy is high. Analogy: cooking dinner before confirming someone is hungry.
7. Hierarchy of MemoriesFast/small/expensive at top, slow/large/cheap at bottom. Caches give illusion of speed + capacity. Analogy: student living close to campus (common destination) not the beach (rare).
8. Dependability via RedundancyInclude redundant components so the system continues when one fails. Analogy: dual rear tires on a truck; drummer's backup stick.

2.3 Below Your Program core

Traces how a high-level program becomes binary machine instructions through layers of abstraction. Each layer hides complexity from the layer above.

Systems SoftwareSoftware providing common services: OS, compilers, loaders, assemblers. Sits between hardware and application software.
Operating SystemManages hardware resources for programs. Handles I/O, memory allocation, protected sharing of the computer among multiple simultaneous applications.
CompilerTranslates high-level language (C, Java) statements into assembly language statements.
AssemblerTranslates symbolic assembly instructions into binary machine language. Ex: ADD A,B → 1000110010100000.
Assembly LanguageSymbolic representation of machine instructions. One line per machine instruction. Human-readable.
Machine LanguageBinary representation of machine instructions — 0s and 1s that hardware actually executes.
Binary Digit (bit)One of two numbers in base 2 (0 or 1). The fundamental unit of computer information.
InstructionA command hardware understands and obeys; represented as a collection of bits.
High-Level LanguagePortable language (C, Java, Python) using words and algebra. Compiled to assembly, then assembled to machine code.

Translation chain

High-level language (C, Java, Python)
         |   Compiler
         v
Assembly language  (symbolic, e.g. LEGv8: ADD X9, X20, X21)
         |   Assembler
         v
Binary machine language  (bits executed directly by CPU)

Example from textbook — C swap function compiled to LEGv8 assembly then assembled to binary:

// C source
swap(int v[], int k) { int temp; temp=v[k]; v[k]=v[k+1]; v[k+1]=temp; }

// LEGv8 assembly (compiler output)
swap: LSL X10,X1,3   ADD X10,X0,X10
      LDUR X9,[X10,0]  LDUR X11,[X10,8]
      STUR X11,[X10,0] STUR X9,[X10,8]
      BR LR

// Binary (assembler output)
00000000101000100000000100011000 ...

Three advantages of high-level languages: (1) natural thinking; (2) higher programmer productivity; (3) machine-independence — recompile the same source for any CPU.

Some compilers skip assembly and produce binary directly. Both paths exist in practice.

2.4 Under the Covers (Hardware) core

Five classic computer components, display technology, processor internals, memory hierarchy, ISA, and networking.

Five classic components (Big Picture — every computer past and present)

InputWrites data to memory. Ex: keyboard, microphone, camera, touchscreen.
OutputReads data from memory for the user. Ex: display, speakers, printer.
MemoryStores instructions and data for running programs.
DatapathPerforms arithmetic operations — the "brawn" of the processor.
ControlCommands the datapath, memory, and I/O according to instructions — the "brain."
Processor (CPU)Datapath + Control combined. Also called Central Processor Unit. The active part of the computer.

Display technology

LCD (Liquid Crystal Display)Does NOT generate light — controls transmission of light through liquid polymer layer between polarizers. Backlit.
Active Matrix DisplayLCD with a transistor switch at each individual pixel for precise current control and sharper images.
PixelSmallest picture element. Color = 8 bits each for R, G, B = 24 bits/pixel = millions of distinct colors. Stored in a frame buffer.

Frame buffer: the bit map in memory that stores the image. Hardware reads it at the refresh rate to drive the display.

Memory types

Integrated Circuit (chip)Device combining dozens to millions of transistors. The building block of modern computers.
DRAMDynamic RAM — main memory. ~50 ns access. Volatile. Cheaper, denser than SRAM. Dominated main memory since 1975.
SRAMStatic RAM — used for cache. Faster, less dense, more expensive per bit than DRAM. Also volatile.
Cache MemorySmall, fast SRAM buffer inside/near CPU. Acts as buffer for slower DRAM. Gives programmer illusion of fast, large memory.
Volatile MemoryLoses data when power is removed. DRAM and SRAM are volatile. Also called "main memory" or "primary memory."
Nonvolatile MemoryRetains data without power. Flash, magnetic disk, DVD. Also called "secondary memory."
Flash MemoryNonvolatile semiconductor. 5–50 μs access. Cheaper than DRAM; more expensive than disk. Wears out after 100K–1M writes. Standard secondary storage for PMDs.
Magnetic Disk (HDD)Rotating platters. 5–20 ms access. Lowest $/GB. Nonvolatile. Dominated server secondary storage.

Memory comparison

TypeVolatile?SpeedCost/GBUse
SRAMYesFastest (sub-ns)HighestCache
DRAMYes~50 nsMediumMain memory

ISA and ABI

Instruction Set Architecture (ISA)Abstract interface between hardware and lowest-level software. Specifies all instructions, registers, memory access, I/O needed to write machine code. Enables many hardware implementations to run identical software.
Application Binary Interface (ABI)ISA + OS interfaces used by application programmers. Defines binary portability across computers.
ImplementationSpecific hardware that obeys an ISA abstraction. Many implementations can share one ISA at different cost/performance.

Big Picture: ISA = the interface (what programmers see). Implementation = the hardware realization. Separating them allows Moore's Law improvements without changing software.

Networking

LAN (Local Area Network)Within a building. Ethernet: up to 1 km, up to 40 Gbps. Interconnected with switches.
WAN (Wide Area Network)Spans continents. Backbone of the Internet. Based on optical fibers leased from telecom companies.

2.5 Technologies for Building Processors & Memory core

Chip manufacturing from silicon ingot to packaged die. Understand yield and why bigger dies cost more per good unit.

TransistorOn/off switch controlled by an electric signal. The fundamental building block of all ICs.
VLSIVery Large-Scale Integrated circuit — hundreds of thousands to millions of transistors on one chip.
SiliconNatural semiconductor (found in sand). Neither conductor nor insulator — can be made to behave as either with doping. Basis of modern chips.
Silicon Crystal IngotRod of pure silicon crystal, 8–12 inches diameter, 12–24 inches long. Starting material for wafer production.
WaferThin slice from the ingot (≤0.1 inches). Undergoes 20–40 chemical processing steps to create patterned transistors, conductors, and insulators.
DieIndividual rectangular section cut from a patterned wafer. Informally called a "chip." Defective dies are discarded.
DefectMicroscopic flaw in wafer or in patterning steps. Can cause a die to fail. Virtually impossible to avoid entirely — hence the concept of yield.
YieldPercentage of good dies from total dies on the wafer. Larger die = lower yield + fewer per wafer = much higher cost per chip.

Manufacturing process (in order)

Silicon ingot
  → Slicer → Blank wafers
  → 20–40 processing steps → Patterned wafers
  → Wafer tester (map defects)
  → Dicer → Tested dies (bad ones discarded)
  → Bond die to package → Packaged dies
  → Part tester (final QC) → Ship to customers

2.6 Performance core

The CPU performance equation is the most tested content in this chapter. Know every term, every formula, and the worked example pattern.

Response Time (Execution Time)Total time to complete a task: disk, memory, I/O, OS overhead — everything. What the user experiences. Also called "wall-clock time" or "elapsed time."
Throughput (Bandwidth)Tasks completed per unit time. Datacenter metric. Adding more processors typically improves throughput without reducing individual response time.
CPU Execution TimeTime the CPU actually spends computing for a task — excludes I/O wait and time running other programs. CPU time = user CPU time + system CPU time.
User CPU TimeCPU time spent in the user's program itself.
System CPU TimeCPU time the OS spends on behalf of the user's program.
Clock Cycle (tick)One discrete time interval in the CPU. Hardware events are synchronized to the clock. Rate measured in Hz (cycles/second).
Clock PeriodLength of one clock cycle = 1 / Clock_rate. Ex: 4 GHz clock → period = 0.25 ns.
CPIClock Cycles Per Instruction — average number of cycles each instruction takes. Varies by program and by instruction mix. CPI is NOT a fixed property of a CPU.
Instruction Count (IC)Number of instructions executed for a program. Same across all implementations of the same ISA (for the same program/compiler).
IPCInstructions Per Clock Cycle = 1/CPI. Some modern processors fetch & execute multiple instructions per cycle, so CPI < 1 is possible.
Instruction MixDynamic frequency of each instruction type across a program. Affects CPI because different instructions have different cycle counts.

Performance relationship

Performance = 1 / Execution_time "X is n times as fast as Y" means: Performance_X / Performance_Y = n Execution_time_Y / Execution_time_X = n

CPU Performance Equation

CPU_time = Instruction_Count × CPI × Clock_Period CPU_time = (Instruction_Count × CPI) / Clock_Rate Clock_Cycles = Instruction_Count × CPI CPU_time = Clock_Cycles / Clock_Rate

Example 1: Relative speed

A runs in 10 s. B runs the same program in 15 s.
Ratio = Time_B / Time_A = 15/10 = 1.5
A is 1.5x as fast as B.

Example 2: Comparing implementations of same ISA

A: clock period = 250 ps, CPI = 2.0
B: clock period = 500 ps, CPI = 1.2
Both execute I instructions.

CPU_time_A = I × 2.0 × 250 ps = 500I ps
CPU_time_B = I × 1.2 × 500 ps = 600I ps
A is faster. Ratio = 600/500 = 1.2x faster.

Example 3: Finding required clock rate for target time

Program: 10 s on A (2 GHz clock). Target: 6 s on B.
B requires 1.2x cycles of A.

Cycles_A = 10 s × 2×10&sup9; = 20×10&sup9; cycles
Cycles_B = 1.2 × 20×10&sup9; = 24×10&sup9; cycles

CPU_time_B = Cycles_B / Rate_B
6 = 24×10&sup9; / Rate_B
Rate_B = 4 GHz

Example 4: Comparing code sequences (different instruction mix)

CPI by class: A=1, B=2, C=3

Code 1: 2A + 1B + 2C  →  cycles = 2×1 + 1×2 + 2×3 = 10
Code 2: 4A + 1B + 1C  →  cycles = 4×1 + 1×2 + 1×3 = 9

Code 2 is faster (9 vs 10 cycles) despite having more instructions (6 vs 5).
Code 2 CPI = 9/6 = 1.5. Code 1 CPI = 10/5 = 2.0.

Only the product IC × CPI × Clock_Period = CPU Time is a reliable measure of performance. Any one or two of the three factors alone can be misleading.

2.7 The Power Wall core

Why clock rates stopped rising around 2004 and why energy efficiency is now the critical design constraint.

Dynamic Energy (CMOS)Energy consumed when transistors switch states (0↔1). Primary energy source in CMOS chips. Depends on capacitive load × V².
Leakage (Static) EnergyCurrent that flows even when transistors are "off." Responsible for ~40% of server chip power today. Cannot be eliminated by turning transistors off.

Power formulas

Energy_per_transition = Capacitive_load × Voltage² Power = Capacitive_load × Voltage² × Frequency_switched Power ∝ V² — reducing voltage 15% cuts power ~28%. Over 20 years: voltage went from 5 V → 1 V → power increase only ~30x despite clock rate increasing ~1000x.

Why clock rates plateaued ~2004

  • Cooling commodity chips hits a practical limit (~100 W for desktop, ~300 W for high-end servers)
  • Further voltage reduction causes transistors to leak current even when "off"
  • Pentium 4 Prescott (2004): 3.6 GHz, 103 W — abandoned due to thermal problems
  • Post-Prescott: Core 2 reverted to lower clock rates + multiple cores

Power calculation example

New processor vs. old:
  Capacitive load: 85% of old
  Voltage: 85% of old (also reduces frequency by 15%)

Power_ratio = (0.85) × (0.85)² × (0.85)
           = 0.85 × 0.7225 × 0.85 ≈ 0.52

New processor uses about half the power.

Energy (joules) is a better metric than power (watts) for efficiency comparisons. A fast program can finish sooner and reduce total energy even if its instantaneous power is higher.

2.8 Uniprocessors to Multiprocessors core

The shift to multicore was forced by the power wall. Since 2002 performance growth slowed from ~52%/year to ~22%/year for single-core processors.

Multicore MicroprocessorSingle chip with multiple processor cores. "Cores" avoids confusion with "processor" = chip. A quadcore chip = 4 cores.
Instruction-Level Parallelism (ILP)Parallelism hidden from the programmer (pipelining, out-of-order execution). Hardware/compiler finds it automatically. Does NOT require code changes.

Parallel programming challenges

  • Scheduling: dividing the application into roughly equal sub-tasks
  • Load balancing: ensuring each core has approximately equal work at the same time
  • Synchronization overhead: time spent coordinating between processors
  • Communication overhead: time spent transferring data between processors

Unlike pipelining (transparent to programmer), multicore requires programmers to explicitly rewrite code to be parallel. This was historically the "third rail" of computer architecture — companies that depended on such behavioral shifts from programmers previously failed.

Performance improvement history

PeriodAnnual perf growthReason
Pre-mid-1980s~25%/yearTechnology-driven (transistor scaling)
Mid-1980s to 2002~52%/yearAdvanced architecture (pipelining, caches, ILP)
2002 to present~22%/yearPower wall, ILP limits, memory latency

2.10 Fallacies & Pitfalls core

These appear on exams as trap/true-false questions. Know all of them cold.

Pitfall: Expecting proportional speedup from a partial improvement

Amdahl's Law: Execution_time_after = (Execution_time_affected / Amount_of_improvement) + Execution_time_unaffected
Example: 100 s program. Multiply = 80 s. Goal: 5x speedup (run in 20 s).

20 = 80/n + (100 − 80)
20 = 80/n + 20
0  = 80/n   →  n = ∞

CONCLUSION: Impossible. Even making multiply infinitely fast only achieves
100/20 = 5x if the other 20 s could also be eliminated. Since they can't,
the maximum speedup from improving multiply alone is 100/20 = 5x only if
n→∞. In practice the floor is 20 s = 5x maximum, unreachable.
Amdahl's LawPerformance enhancement from improving a feature is limited by how much time that feature is actually used. Quantitative version of the law of diminishing returns. Also limits parallelism speedup.

Fallacy: Idle servers use little power

Google's WSC runs at 10–50% utilization most of the time. A well-tuned server still uses ~33% of peak power at 10% load. Power consumption is not proportional to utilization. This motivates "energy-proportional computing."

Fallacy: Performance and energy efficiency are unrelated

A faster program uses less total energy if it finishes sooner — the entire system draws power during execution. Hardware/software optimizations that reduce time can save energy even if instantaneous power is slightly higher.

Pitfall: Using MIPS as a performance metric

MIPS = Instruction_count / (Execution_time × 10&sup6;) = Clock_rate / (CPI × 10&sup6;)

Three fundamental problems:

  • Cannot compare different ISAs — instruction capability differs, so raw counts are meaningless across architectures
  • MIPS varies by program on the same CPU — CPI changes with instruction mix, so there is no single MIPS rating for a processor
  • MIPS can improve while performance worsens — a program with more instructions each faster can have higher MIPS but longer execution time
MIPS (Million Instructions Per Second)Instruction execution rate. Higher MIPS does NOT reliably mean faster execution. Execution time is the only valid and unimpeachable measure of performance.

Memory hierarchy (fast → slow)

  • Registers — inside CPU, fastest, volatile
  • Cache (SRAM) — fast, small, volatile
  • Main Memory (DRAM) — ~50 ns, volatile
  • Flash — 5–50 μs, nonvolatile, wears out
  • Magnetic Disk — 5–20 ms, nonvolatile, cheapest

Critical distinctions

  • Response time vs. throughput: faster CPU improves both; more CPUs mainly improve throughput
  • Elapsed time vs. CPU time: elapsed includes I/O wait; CPU time does not
  • TB (10¹²) vs. TiB (2&sup4;&sup0;): TiB ~10% larger
  • DRAM (volatile) vs. flash/disk (nonvolatile)
  • ISA (interface) vs. Implementation (hardware)
  • MIPS is flawed — execution time is the only valid measure
  • Amdahl: speedup bounded by the unimproved fraction
  • LCD controls light, doesn't generate it

Chip manufacturing

  • Ingot → Wafer → Patterning (20–40 steps)
  • Wafer test → Dicing into dies → Packaging
  • Yield = good dies / total dies on wafer
  • Bigger die → fewer per wafer + lower yield → higher cost
  • Higher volume → lower cost (mask amortization + process tuning)

Power wall facts

  • Power ∝ V² → 15% voltage cut ≈ 28% power cut
  • Voltage dropped 5 V → 1 V over 20 years
  • Leakage = ~40% of server chip power
  • Clock rates peaked ~2004 (thermal limits)
  • Post-2002 perf growth: ~22%/year (was 52%)
  • Idle server still uses ~33% of peak power at 10% load

Self-Test (Exam-Style Q&A)

Computing Classes & Intro (6 questions)
What are the three traditional classes of computers and what distinguishes each?
Personal computers (single user, cost-performance balance, third-party software), Servers (multiple simultaneous users, network access, throughput + dependability), Embedded computers (single predetermined application, runs in another device, largest class by count). Post-PC adds PMDs (mobile) and cloud/WSC.
What is a supercomputer and how does it relate to servers?
A supercomputer is the highest-performance, highest-cost computer — configured as servers but with tens of thousands of processors and many terabytes of memory, costing tens to hundreds of millions of dollars. Used for weather, oil exploration, protein structure. Represents a tiny fraction of total server market revenue.
What distinguishes a PMD from a PC?
PMDs are battery-operated, wirelessly connected, use touch/speech input instead of keyboard/mouse, and install software as downloaded "apps." PCs use plug-in power and install full binary software. Smartphones passed PCs in annual units sold in 2011.
What is SaaS? Give examples.
Software as a Service — software and data delivered over the Internet via a thin client (browser), with no binary installed locally. Examples: web search, social networking, email services.
What is a terabyte vs. a tebibyte?
Terabyte (TB) = 10¹² bytes (decimal). Tebibyte (TiB) = 2&sup4;&sup0; bytes (binary). TiB is about 10% larger. The binary notation resolves historical ambiguity about which definition of "terabyte" was meant.
A family's home computer used for email and movies is which class? A cardiac pacemaker computer?
Home computer = PC. Cardiac pacemaker computer = embedded (runs one predetermined application inside another device).
Eight Great Ideas (6 questions)
Name all eight great ideas in order.
1. Design for Moore's Law, 2. Abstraction to simplify design, 3. Make the common case fast, 4. Performance via parallelism, 5. Performance via pipelining, 6. Performance via prediction, 7. Hierarchy of memories, 8. Dependability via redundancy.
What does Moore's Law state and what does it mean for design timing?
IC resources double every 18–24 months (Gordon Moore, 1965). Designers must target where technology will be when the project finishes, not when it starts — like a skeet shooter leading the target.
Soccer player runs to where ball will be → which idea? Dual truck tires → which idea?
Soccer player = Design for Moore's Law. Dual tires = Dependability via redundancy.
One person washes dishes, another dries each dish immediately after washing = which idea? Both washing and drying at the same time = which idea?
One washes, one dries immediately = Pipelining (overlapping stages). Both working simultaneously on different dishes = Parallelism.
Why is "make the common case fast" often more effective than optimizing the rare case?
The common case executes far more frequently, so a modest speedup there yields larger total gains. Also, the common case is often simpler and easier to optimize. Requires measurement to identify what is actually common.
Which great idea explains why caches work and why we use them?
Hierarchy of memories — small, fast, expensive memory (SRAM cache) at the top provides speed, while large, slow, cheap memory (DRAM, disk) provides capacity. Together they give the programmer the illusion of fast + large + cheap.
Below Your Program (5 questions)
What is the full translation chain from C source code to machine execution?
C source → Compiler → Assembly language → Assembler → Binary machine language → CPU executes. (Some compilers skip assembly and go directly to binary.)
What is the difference between assembly language and machine language?
Assembly language uses symbolic mnemonics (ADD A,B) — human-readable. Machine language is binary (1000110010100000) — directly executed by hardware. The assembler translates one to the other, one instruction at a time.
Which of these is a high-level language instruction? 00000000101 / ADD X2,X4,X2 / temp=v[k];
temp=v[k]; is high-level (C). ADD X2,X4,X2 is assembly. 00000000101 is machine language (binary).
What three advantages do high-level languages have over assembly?
1. More natural thinking (words/algebra instead of cryptic symbols). 2. Greater programmer productivity (fewer lines for same logic). 3. Machine-independence (same source runs on different CPUs after recompiling).
What does an operating system do?
Manages hardware resources for application programs: handles basic I/O, allocates memory and storage, provides protected sharing of the computer among multiple simultaneous applications. Examples: Linux, iOS, Windows.
Hardware Components (10 questions)
What are the five classic components of a computer?
Input, Output, Memory, Datapath, Control. Datapath + Control = Processor/CPU. This organization applies to every computer past and present.
What is the role of Control vs. Datapath in a processor?
Datapath performs arithmetic operations ("brawn"). Control tells the datapath, memory, and I/O what to do according to the program's instructions ("brain").
Does an LCD display generate light? How does it work?
No. An LCD controls the transmission of light from a backlight through liquid crystal molecules that twist or straighten based on applied charge. Light passes when bent (no charge), blocked when straightened (charge applied) — combined with polarizing filters.
How many bits represent one color pixel, and how is color encoded?
24 bits per pixel: 8 bits each for red, green, and blue. This allows ~16 million distinct colors. The image is stored in a frame buffer in memory.
What is the difference between DRAM and SRAM?
DRAM: slower (~50 ns), denser, cheaper per bit — used for main memory. SRAM: faster (sub-ns), less dense, more expensive per bit — used for cache. Both are volatile.
What is the difference between volatile and nonvolatile memory? Give one example of each.
Volatile loses data without power (DRAM, SRAM). Nonvolatile retains data (flash memory, magnetic disk). Main memory = volatile; secondary memory = nonvolatile.
What is flash memory's key limitation that doesn't affect DRAM or magnetic disk?
Flash bits wear out after 100,000 to 1,000,000 write cycles. File systems must track write counts and spread writes to avoid wearing out specific locations ("wear leveling").
What is an ISA, and why is it important?
Instruction Set Architecture — the abstract interface between hardware and lowest-level software. Specifies all instructions, registers, memory access, I/O needed to write machine language. Enables multiple hardware implementations to run identical software at varying cost/performance.
What is the difference between an ISA and an implementation?
ISA = the abstract interface (what programmers see). Implementation = the specific hardware that realizes the ISA. Many implementations can share one ISA. Example: many ARM chip generations share the ARM ISA.
Ethernet is an example of which type of network? What bandwidth can it reach?
LAN (Local Area Network). Ethernet can reach up to 40 Gbps. WANs span continents via optical fiber and form the Internet backbone.
Chip Manufacturing (4 questions)
What is the manufacturing sequence from silicon to a packaged chip?
Silicon crystal ingot → Slicer → Blank wafers → 20–40 processing steps → Patterned wafers → Wafer tester → Dicer (cuts dies) → Bond good dies to packages → Part tester → Ship to customers.
What is a die, and what is yield?
A die is an individual rectangular section cut from a patterned wafer (informally: a "chip"). Yield = percentage of good dies / total dies on wafer. Defects make some dies fail — only good dies are packaged and sold.
Why does a larger die cost more per unit than a smaller die?
Two compounding reasons: (1) fewer dies fit on a fixed-size wafer, and (2) larger dies have more area and thus higher probability of containing a defect — lower yield. Both effects sharply increase cost per good die.
Why does higher production volume lower chip cost?
Expensive one-time mask costs are amortized over more units; engineering development costs spread over more units; the manufacturing process can be tuned to the specific design, raising yield.
Performance Equations (10 questions)
Computer A runs a program in 10 s, Computer B in 15 s. How much faster is A?
Perf_A/Perf_B = Time_B/Time_A = 15/10 = 1.5. A is 1.5 times as fast as B.
Write the CPU performance equation in two equivalent forms.
CPU_time = IC × CPI × Clock_period. Equivalently: CPU_time = (IC × CPI) / Clock_rate.
Computer A: 250 ps clock, CPI 2.0. Computer B: 500 ps clock, CPI 1.2. Same ISA. Which is faster and by how much?
Time_A = I×2.0×250 = 500I ps. Time_B = I×1.2×500 = 600I ps. A is faster. A is 600/500 = 1.2x faster than B.
Program runs in 10 s on A (2 GHz). Want 6 s on B which needs 1.2x the cycles of A. What clock rate must B have?
Cycles_A = 10×2×10&sup9; = 20×10&sup9;. Cycles_B = 1.2×20×10&sup9; = 24×10&sup9;. Rate_B = 24×10&sup9;/6 = 4 GHz.
CPI per class: A=1, B=2, C=3. Code 1: 2A+1B+2C. Code 2: 4A+1B+1C. Which is faster? What is each CPI?
Cycles_1 = 2+2+6 = 10. Cycles_2 = 4+2+3 = 9. Code 2 is faster despite more instructions (6 vs 5). CPI_1 = 10/5 = 2.0. CPI_2 = 9/6 = 1.5.
What is the difference between elapsed time and CPU time?
Elapsed (wall-clock) time includes everything: I/O waits, OS overhead, other programs running. CPU time includes only cycles the CPU spent on your task. CPU time = user CPU time + system CPU time.
If CPI increases 20% and clock cycle time decreases 10%, does execution time increase or decrease?
New time = IC × (1.2×CPI) × (0.9×Period) = 1.08 × original. Execution time increases by 8%.
A new compiler reduces IC to 0.6x but increases CPI by 1.1x. Original time: 15 s. New time?
New time = 15 × 0.6 × 1.1 = 9.9 s.
Can CPI ever be less than 1? What is it called?
Yes. Some processors fetch and execute multiple instructions per clock cycle. If average is 2 instructions/cycle, CPI = 0.5. The inverse (2) is called IPC (instructions per clock cycle).
Why can't you compare performance just by looking at instruction count?
Execution time = IC × CPI × Clock_period. Fewer instructions may have higher CPI and thus run slower. All three factors must be considered together.
Power, Multicore, Benchmarks, Fallacies (11 questions)
What is CMOS dynamic power and what does it depend on?
Energy consumed when transistors switch states (0↔1). Power = Capacitive_load × Voltage² × Switching_frequency. Power scales with voltage squared.
Why did clock rates stop increasing around 2004?
Power hit a practical cooling limit for commodity chips (~100 W). Further voltage reduction causes transistor leakage (static power — ~40% of server chip power today). Pentium 4 Prescott (103 W, 2004) was abandoned due to thermal problems.
If voltage is reduced by 25%, by what factor does dynamic power change?
Power ∝ V². New power = (0.75)² = 0.5625 of original — approximately a 44% reduction.
Why is energy (joules) a better metric than power (watts) for efficiency comparisons?
Power is joules/second — it says nothing about how long the task takes. Total energy = power × time. A faster, slightly power-hungry processor may consume less total energy than a slow, low-power one because the whole system draws power during execution.
What forced the switch from uniprocessors to multicore chips?
The power wall — increasing clock rate requires proportionally more power, which hits thermal limits. Multiple cores at lower clock rates achieve higher throughput without exceeding the power budget.
What is Amdahl's Law and what does it imply about the limits of improvement?
Performance enhancement from improving a feature is bounded by how much time that feature is used. If 20% of runtime can't be improved, the maximum speedup from any enhancement is 1/0.2 = 5x, regardless of how much you improve the other 80%.
A program runs 100 s. Multiply = 80 s. You improve multiply by 4x. What is the new execution time and speedup?
New time = 80/4 + 20 = 20 + 20 = 40 s. Speedup = 100/40 = 2.5x.
What is a SPECratio and why is the geometric mean used for the overall score?
SPECratio = reference_time / measured_time. Bigger = faster. Geometric mean is used so the overall score gives the same relative ranking regardless of which reference machine is chosen (arithmetic mean does not have this property).
Why is MIPS a misleading performance metric? Give all three reasons.
1. Can't compare different ISAs (instructions differ in capability). 2. MIPS varies program-to-program on the same CPU (CPI varies with instruction mix). 3. Higher MIPS can accompany worse performance (more instructions each faster can yield lower execution time with higher MIPS).
A server runs at 10% utilization. Does it use about 10% of peak power?
No. Even a specially tuned server uses ~33% of peak power at 10% load. Real-world servers that aren't tuned for this benchmark do even worse. This is why "energy-proportional computing" is a design goal, not a current reality.
Does designing for better performance always hurt energy efficiency?
No. This is a fallacy. A faster program finishes sooner, reducing the time the entire system consumes power. Hardware/software optimizations that reduce execution time can save total energy even if instantaneous power is slightly higher.

Common Exam Traps

  • "An LCD display generates its own light." No. An LCD controls the transmission of light from a separate backlight — it does not emit light.
  • "A terabyte equals a tebibyte." No. TB = 10¹² bytes (decimal). TiB = 2&sup4;&sup0; bytes (binary). TiB is about 10% larger than TB.
  • "Cache memory uses DRAM because it needs to be fast." No. Cache uses SRAM. DRAM is slower — it's used for main memory. SRAM is faster but more expensive and less dense.
  • "Flash memory is nonvolatile and never degrades." No. Flash is nonvolatile, but bits wear out after 100,000–1,000,000 write cycles. File systems must use wear leveling to manage this.
  • "Pipelining requires programmers to rewrite code." No. Pipelining (instruction-level parallelism) is transparent to programmers — the hardware/compiler handles it. Multicore parallelism, however, does require explicit programmer effort.
  • "The computer with fewer instructions executed always runs faster." Not necessarily. Fewer instructions with higher CPI can run slower. Execution time = IC × CPI × Clock_period. All three factors matter.
  • "MIPS is a reliable way to compare computer performance." No. MIPS varies by program on the same CPU, can't compare across ISAs, and can be inversely correlated with actual performance. Execution time is the only valid measure.
  • "Improving multiply by 5x will make a program 5x faster if multiply uses 80% of runtime." No. By Amdahl's Law: new time = 80/5 + 20 = 36 s (was 100 s) → only 100/36 ≈ 2.78x speedup.
  • "A server at 10% utilization uses about 10% of peak power." No. Real servers use ~33% of peak power even at 10% load. Power is not proportional to utilization.
  • "Better performance always means worse energy efficiency." No. Faster execution means less total time the whole system draws power. Performance and energy efficiency can both improve together.
  • "ISA and implementation are the same thing." No. ISA is the abstract interface (what programmers write to). Implementation is the specific hardware realizing the ISA. Multiple implementations at different cost/performance can share one ISA.
  • "Replacing a uniprocessor with multicore always improves response time for any program." No. Multicore primarily improves throughput. A single-threaded sequential program does not benefit unless it is explicitly rewritten to use multiple cores.
  • "Embedded computers are just small PCs." No. Embedded computers run one predetermined application, are integrated into another device, are optimized for cost/power in a specific context, and often have no visible OS to users. They are not general-purpose.
  • "A program with a higher MIPS rating always runs faster." No. If the program with more MIPS executes more (but simpler) instructions with shorter clock cycles, MIPS can be higher while execution time is longer. Example: same task, one program uses 8 billion fast instructions, another uses 10 billion even faster ones — check all three factors.
