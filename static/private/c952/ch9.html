9.1 Introduction

Two big distinctions to anchor everything else:

Combinational logicNo memory. Output depends only on current inputs. Same inputs → same outputs. Examples: AND/OR gates, ALU, decoders, multiplexors.
Sequential logicContains memory (state). Output depends on inputs AND current state. Examples: register file, FSM, the entire processor.
Asserted signal(Logically) true, value 1.
Deasserted signal(Logically) false, value 0.

Two voltage levels (high/low) → 1/0; intermediate voltages happen only briefly during transitions. We talk about asserted/deasserted instead of voltage to avoid ambiguity across logic families.

9.2 Gates, Truth Tables, Logic Equations core

Boolean Operators

OperatorNotationResult
AND (logical product)A · B or A & B1 iff both inputs are 1
OR (logical sum)A + B or A | B1 if either input is 1
NOT (inverter)Ā or ~A1 iff input is 0
XORA ⊕ B or A ^ B1 iff exactly one input is 1

Boolean Algebra Laws (for manipulation/minimization)

LawForm
IdentityA + 0 = A · 1 = A
Zero / OneA + 1 = 1; A · 0 = 0
InverseA + Ā = 1; A · Ā = 0
CommutativeA + B = B + A; A · B = B · A
Associative(A + B) + C = A + (B + C); same for ·
DistributiveA · (B + C) = AB + AC; A + BC = (A+B)(A+C)
DeMorgan~(A + B) = Ā · B̄; ~(A · B) = Ā + B̄

Truth Tables & Gates

  • A truth table fully describes any combinational logic: 2n rows for n inputs.
  • "Bubbles" on a gate input or output indicate inversion (saves drawing inverters).
  • NAND = inverted AND; NOR = inverted OR. Either is universal — any logic function can be built from a single gate type if it is inverting.
A 2-input multiplexor is also universal — it can build NAND or NOR.

9.3 Combinational Logic core

Common Building Blocks

Decodern-bit input → 2n outputs. Exactly one output is asserted, corresponding to the binary value of the input. A "3-to-8 decoder" has 3 inputs and 8 outputs.
EncoderInverse: 2n inputs → n-bit output.
Multiplexor (mux)Selector. n data inputs + ⌈log₂n⌉ select bits → 1 output. Internally: decoder + AND array + OR. 2-input mux: C = (Ā · S̄ · A) ... = A if S=0, else B.
BusMulti-bit signal treated as one logical line. A 64-bit mux is built from 64 1-bit muxes sharing a single select. Drawn as a thicker line.

Two-Level Logic

Any logic function can be written in canonical form with at most 2 levels of gates: AND followed by OR (sum-of-products / SOP) or OR followed by AND (product-of-sums / POS). Inversions appear only on individual variables.

PLA (Programmable Logic Array) key

  • Two-stage structured implementation of SOP.
  • AND plane — each row is a product term (also called a minterm) made from inputs and their inverses.
  • OR plane — each output is the OR of selected product terms.
  • Total size ∝ (#inputs × #product terms) + (#outputs × #product terms).
  • PLA is sparse: only the product terms that are actually true contribute.

ROM (Read-Only Memory)

  • Height = 2m entries (m address lines). Width = output bits per entry. Total bits = height × width. "Shape" = both.
  • Variants: PROM (one-time programmable), EPROM (UV-erasable).
  • Encodes the truth table directly: address = inputs, contents = outputs.
  • Fully decoded — every input combination has an entry. PLA is partially decoded — only used product terms have entries.
  • ROMs scale exponentially with input count; PLAs grow with the number of distinct product terms (usually much slower).
  • ROM advantage: any function with the same I/O dimensions fits, easy to update by changing contents.

Don't Cares key

TypeMeaning
Output don't careX in output column. Output value is irrelevant for that input combination — minimization tools may set it to whatever shrinks the equation.
Input don't careX in input column. The output doesn't depend on that input for the row.

Specifying don't-cares is essential for efficient logic. Karnaugh maps (K-maps) help with hand minimization but are impractical beyond ~5 variables — use a CAD tool (e.g., Espresso).

9.4 Hardware Description Language (Verilog)

Modern digital design uses HDLs for both simulation and logic synthesis. Two main HDLs: Verilog (C-flavored, more common in industry) and VHDL (Ada-flavored).

Spec Styles

BehavioralDescribes what the system does (functionality). Synthesis tools generate gates from this.
StructuralDescribes how the system is organized (gates, modules, interconnects).

Verilog Datatypes

TypeUse
wireCombinational signal — can only be set with continuous (assign) statements.
regHolds a value (may or may not become an actual register). Set inside always blocks.
Values0, 1, X (unknown), Z (high impedance for tri-state).

Constant Notation

4'b0100   // 4-bit binary 0100  (= 4)
4'd4      // 4-bit decimal 4
8'h F0    // 8-bit hex F0
{32{2'b01}}  // replicate 32 times → 64-bit pattern 010101...
{A[31:16], B[15:0]}  // concatenate

Module Body Constructs

  • initial — initialize regs at startup.
  • Continuous assign — combinational logic only. wire targets only.
  • always @(...) — block re-evaluated when sensitivity-list signals change. Can produce combinational or sequential logic depending on construction.
  • Module instances — hierarchical composition.

Blocking vs Nonblocking Assignment key

FormBehaviorUse for
= (blocking)Like C: RHS evaluated, LHS assigned, then next statement runs.Combinational logic inside always
<= (nonblocking)All RHSs in the block evaluated first, then all LHSs assigned simultaneously.Sequential logic (e.g., flip-flop updates on clock edge)

Avoiding Accidental Sequential Logic

  • Place all combinational logic in assign or in an always block whose sensitivity list includes all input signals.
  • Every path through an always block must assign the same set of bits — otherwise the synthesizer infers a latch.

Edge-Triggered Register in Verilog

always @(posedge clock) A <= b;   // A is a reg, written on rising clock edge

9.5 Constructing a Basic ALU core

Build a 64-bit ALU from 64 connected 1-bit ALUs.

1-Bit Adder ("Full Adder", 3 inputs / 2 outputs)

CarryOut = (a · b) + (b · CarryIn) + (a · CarryIn) Sum = (a · ~b · ~CarryIn) + (~a · b · ~CarryIn) + (~a · ~b · CarryIn) + (a · b · CarryIn) = a ⊕ b ⊕ CarryIn (using XOR)

A full adder has 3 inputs and 2 outputs (3,2). A half-adder has 2 inputs and 2 outputs (2,2).

1-Bit ALU Build-Up

  1. Logical unit: AND gate + OR gate, mux selects with control Operation.
  2. Add full adder. 4-to-1 mux selects AND / OR / Add (and a 0 input for utility).
  3. Subtract: add a 2:1 mux to invert b (Binvert). Set CarryIn=1 on least-significant bit. Result = a + ~b + 1 = a − b. (This is why two's complement is universal: cheap subtraction.)
  4. NOR: by DeMorgan, ~(a + b) = ~a · ~b. Add an Ainvert mux too — pass ~a and ~b through the AND.
  5. CBZ pass: add a "Pass" input to the operation mux that just hands through one operand (for compare-zero).

64-Bit ALU = 64 1-Bit ALUs (Ripple-Carry)

  • CarryOut of bit i feeds CarryIn of bit i+1 — slow, but simple.
  • Adds a Zero detect (NOR of all 64 result bits) to support equality compare and CBZ.
  • Most-significant bit's adder has a direct "Set" output for set on less than.

ALU Control Lines

Combination of Ainvert, Bnegate (Binvert + CarryIn=1), and 2-bit Operation = 4 control bits selecting AND / OR / add / subtract / SLT / NOR / pass.

Shifts (1–63 bits) are typically done outside the ALU in a barrel shifter, which performs any shift in roughly the time of one add.

9.6 Faster Addition: Carry Lookahead core

Ripple carry is too slow for wide adders — every bit waits for the previous bit's carry. Speed up by computing carries in parallel.

Generate & Propagate key

g_i = a_i · b_i // bit i generates a carry regardless of input p_i = a_i + b_i // bit i propagates an incoming carry c_{i+1} = g_i + p_i · c_i

Domino analogy: a far-away "generate" can tip the chain provided every "propagate" between it and the target is true.

4-Bit CarryIn Equations (level 1 abstraction)

c1 = g0 + p0·c0
c2 = g1 + p1·g0 + p1·p0·c0
c3 = g2 + p2·g1 + p2·p1·g0 + p2·p1·p0·c0
c4 = g3 + p3·g2 + p3·p2·g1 + p3·p2·p1·g0 + p3·p2·p1·p0·c0

Two-Level Carry Lookahead (16-bit using 4 × 4-bit blocks)

P0 = p3·p2·p1·p0 // group propagate G0 = g3 + p3·g2 + p3·p2·g1 + p3·p2·p1·g0 // group generate C_{i+1} = G_i + P_i · C_i

The "super" propagate/generate signals plug into the same lookahead formula, one level higher. This is how the book builds a 16-bit CLA from 4 × 4-bit CLA blocks plus a top-level lookahead unit.

Speed Comparison (count gate delays)

Adder type16-bit gate delays for carry-in → carry-out
Ripple carry16 × 2 = 32
Two-level carry lookahead2 (for p,g) + 2 (for P,G) + 1 ≈ 5 — roughly 6× faster

9.7 Clocks core

A clock is a free-running square wave with fixed cycle time. Clock frequency = 1 / clock period.

Edge-Triggered Clocking

  • State changes happen only on a clock edge (rising or falling). The book uses edge-triggered throughout.
  • Sampling on an edge is essentially instantaneous → no race condition from "almost-the-same-time" sampling.
  • Clock period must be ≥ longest combinational path so all signals settle before the next edge.
  • Allows reading and writing the same state element in one cycle (e.g., register file) without race.

A synchronous system uses a clock and only reads signals when the clock indicates they are stable.

9.8 Memory Elements: Flip-Flops, Latches, Registers core

S-R Latch (unclocked)

Cross-coupled NOR (or NAND) pair. S sets, R resets, both deasserted holds. Asserting both at once is illegal (oscillation/metastability).

D Latch vs D Flip-Flop key

ElementState changes when…Sometimes called
D LatchClock is asserted (level-sensitive). While clock is high, output Q tracks D. "Transparent latch."Open / closed
D Flip-FlopOnly on the clock edge. Built from two D latches (master/slave) so output changes only when clock transitions.Edge-triggered

Setup & Hold Times

Setup time (tsetup)Minimum time the input must be valid before the clock edge.
Hold time (thold)Minimum time the input must remain valid after the clock edge.

Hold times are usually 0 or near-zero on modern flip-flops, so setup is the main concern.

Verilog D Flip-Flop

module DFF(clock, D, Q, Qbar);
  input clock, D;
  output reg Q;
  output Qbar;
  assign Qbar = ~Q;
  always @(posedge clock)
    Q = D;
endmodule

Register File key

  • Set of registers indexed by a register number.
  • For read: a decoder + multiplexor combination per port. (For two read ports: two n-to-1 muxes, each 64 bits wide.)
  • For write: a decoder converts the register number into a one-hot enable that gates the clock to the chosen register only.
  • Reading is combinational; writing is sequential — that's why in Verilog the read is a continuous assign and the write is in an always @(posedge clock) block.
  • If you read and write the same register in one cycle, you get the OLD value (write happens at edge); to "forward" the new value, extra logic is required.

9.9 SRAMs & DRAMs core

SRAM

  • Static — value held by cross-coupled inverters as long as power is on. 4–6 transistors per bit.
  • Single access port (read OR write). Fixed access time.
  • Configuration: "4M × 8" = 4M addressable entries, each 8 bits. Height = entries, width = bits per entry.
  • Control signals: Chip Select, Output Enable (for reads), Write Enable (for writes).
  • Typical 2004 read access time: 2–4 ns (small) up to 8–20 ns (large, ~32Mb).
  • Built using a two-level decode (rectangular array) + bit lines driven by tri-state buffers from each cell. A tri-state buffer can be asserted, deasserted, or high-impedance — only one driver per shared line at a time.

DRAM

  • Dynamic — value stored as charge on a capacitor accessed by 1 transistor (the pass transistor). 1 transistor per bit → much denser/cheaper than SRAM.
  • Charge leaks → must be refreshed periodically (every few ms).
  • Refresh = read row, write same value back. Whole row refreshed in one cycle. Refresh consumes ~1–2% of cycles.
  • Two-level addressing: RAS (Row Access Strobe) selects a row → all column bits latched; CAS (Column Access Strobe) selects bit(s) from latched row. Same address pins reused for row and column to save package cost.
  • Read works by precharging the bit line halfway and detecting the small voltage swing with a sense amplifier.
  • Typical 2004 access time: 45–65 ns (5–10× slower than SRAM).
  • Use: DRAM = main memory (cheap per bit). SRAM = caches and registers (fast).

SDRAM & DDR

  • SDRAM (synchronous DRAM): clock-driven burst transfers from sequential addresses within a row. Burst = starting address + length; subsequent bits clocked out without re-supplying address.
  • DDR RAM: transfers on both rising and falling edges → 2× bandwidth.

Error Detection & Correction

CodeCapabilityNotes
1-bit parityDetect any single-bit error (or odd # of errors)Distance-2 code. Cannot correct, cannot reliably detect 2 errors.
SEC (single-error correcting)Correct 1-bit errorDistance-3 code. Uses Hamming code.
SEC-DED (1 correct, 2 detect)Correct 1, detect 2Distance-4 code. Typical: 7 extra bits per 64 data bits, 8 extra per 128.

9.10 Finite-State Machines (FSMs) core

Sequential systems are described by an FSM: a set of states, a next-state function (current state + inputs → next state), and an output function. With n state bits there are 2n possible states.

Moore vs Mealy key

Moore machineOutputs depend ONLY on current state. May need more states. Faster (output is a function of a stable state register).
Mealy machineOutputs depend on current state AND inputs. Often fewer states. Output can change asynchronously with input.

Moore and Mealy are equivalent in capability — one can mechanically convert between them. The book uses Moore for the processor controller and a Mealy version for the cache controller.

Designing an FSM

  1. Decide states, inputs, outputs.
  2. Build next-state table (state × input → next state).
  3. Build output table (state → outputs for Moore; state × input → outputs for Mealy).
  4. Draw the state diagram (nodes = states, arc labels = input conditions, node interior = active outputs).
  5. State assignment: assign binary numbers to states (clever choices reduce logic).
  6. Implement: state register + combinational logic for next-state and output functions (often a PLA or ROM).

Example: 2-State Traffic Light

  • States: NSgreen, EWgreen.
  • Inputs: NScar, EWcar.
  • Outputs: NSlite, EWlite.
  • Rule: stay in current state if no car waiting in opposite direction; switch otherwise.

9.11 Timing Methodologies

Edge-Triggered Clock-Period Constraint

T_clock ≥ t_prop(FF) + t_combinational(max) + t_setup(FF) ( + t_skew if there is clock skew )

Here tprop is the flip-flop's clock-to-Q delay. For correctness against skew, allow extra margin equal to the maximum skew between any two state elements.

Clock skewDifference in absolute time when two state elements see the clock edge. Caused by different routing path lengths.
RaceWhen the contents of a state element depend on relative speeds of logic elements. Skew + short combinational path can let an output reach the next FF before that FF's clock edge arrives → incorrect operation.

Level-Sensitive Timing & Two-Phase Clocking

  • Alternative to edge-triggered. State changes occur while a clock level is high or low — not instantaneous.
  • Use two non-overlapping clock phases (φ₁, φ₂) — at most one is asserted at a time.
  • Alternate latches between phases: a φ₁-latch outputs into combinational logic, then a φ₂-latch captures. Non-overlap protects against race.

Asynchronous Inputs & Synchronizers key

MetastabilityIf a flip-flop input violates setup/hold, the output may settle in the indeterminate region between high and low for an unbounded time.
Synchronizer failureWhen metastable output is read by downstream logic; some gates see 0, others see 1 → divergent state.
Synchronizer (single FF)Naive sample of an asynchronous signal with one D-FF — exposed to metastability.
Two-FF synchronizerReliable in practice: a metastable output of FF1 has time to resolve before FF2 samples it on the next clock edge. Probability of failure becomes vanishingly small (years between failures).

9.12 Field Programmable Devices (FPDs)

For designs that don't justify a custom IC, FPDs let you configure off-the-shelf chips to implement combinational and sequential logic.

FPD Family

TypeDescription
PLD (Programmable Logic Device)Combinational only.
SPLD (Simple PLD)Single PLA or PAL. PAL = programmable AND-plane + fixed OR-plane.
CPLD (Complex PLD)Multiple logic blocks + configurable interconnect.
FPGA (Field-Programmable Gate Array)2-D array of cells (combinational logic + flip-flops) connected by configurable routing channels. Cells = LUTs (lookup tables) — small RAMs that act as truth tables. Modern FPGAs include adders, RAM blocks, even RISC cores.

Configuration Mechanisms

  • Antifuse: programmed once at programming time → permanent connection.
  • SRAM-based: downloads configuration on power-on. Volatile, slightly higher resistance, but reconfigurable.

In FPGAs, ~90% of die area is interconnect, only ~10% is logic and memory blocks.

Self-Test Q&A

Combinational vs sequential, gates (6)
Define combinational logic.
Logic with no memory — output depends only on current input. Same input → same output, every time.
Define sequential logic.
Logic that contains state (memory). Output depends on current input AND stored state.
What does "asserted" mean?
Logically true (1). "Deasserted" means logically false (0). The terms abstract away which voltage encodes which.
Why are NAND and NOR called universal?
Any logic function can be built from a single inverting gate type. All other gates derive from these.
A 2-input mux is also universal — sketch how to build NOT.
Wire S=A, input 0 = 1, input 1 = 0; output is ~A.
How many rows in a truth table for n inputs?
2n.
Boolean algebra & PLAs (6)
State DeMorgan's two laws.
~(A + B) = ~A · ~B and ~(A · B) = ~A + ~B.
What is sum-of-products form?
A logical OR (sum) of product (AND) terms. The canonical 2-level form (after individual variable inversions) for any logic function.
What's a minterm?
A product term in the AND plane of a PLA — a logical AND of inputs and/or their inverses corresponding to one truth-table row.
PLA vs ROM — which is more efficient for sparse logic and why?
PLA. A ROM contains an entry for every possible input combination (2m), even if most go unused. A PLA encodes only the minterms that are actually needed.
What does ROM stand for, and what are PROM and EPROM?
Read-Only Memory. PROM is a one-time-programmable ROM. EPROM is erasable (UV light).
What are output don't-cares useful for?
They let logic minimization tools choose 0 or 1 for that input combination, which can collapse multiple truth-table rows together and shrink the gate network.
Decoders, multiplexors, buses (4)
A 3-to-8 decoder — what's at its outputs?
8 outputs; exactly one is asserted, corresponding to the binary value of the 3-bit input. All others are 0.
How is a multiplexor internally structured for n data inputs?
Decoder generating n one-hot signals from log₂n select bits, AND each one-hot with the corresponding data input, OR the AND outputs.
A 64-bit-wide multiplexor selecting between two 64-bit buses — how many 1-bit muxes is that?
64, all sharing the same select signal.
What is a "bus" in this context?
A collection of data lines treated as a single logical signal, drawn as a thicker line in diagrams.
1-bit ALU and full adder (5)
What does a (3,2) adder mean?
A full adder — 3 inputs (a, b, CarryIn) and 2 outputs (Sum, CarryOut).
Write CarryOut as a Boolean equation.
CarryOut = a·b + b·Cin + a·Cin.
Write Sum using XOR.
Sum = a ⊕ b ⊕ CarryIn.
How does the ALU compute a − b?
Asserts Binvert (so input 2 becomes ~b), sets the LSB CarryIn = 1; the adder computes a + ~b + 1 = a − b.
Why does this trick make two's complement universally cheap?
Subtraction reuses the existing adder hardware — no separate subtractor needed.
Carry lookahead (5)
Define generate g_i and propagate p_i.
g_i = a_i · b_i (carries out regardless of input). p_i = a_i + b_i (passes incoming carry through).
Write c_{i+1} in terms of g and p.
c_{i+1} = g_i + p_i · c_i.
What's a "super propagate" P_i?
AND of all the bit-level propagates in a group: P_i = p3·p2·p1·p0 — the group passes a carry only if every bit propagates.
Approximate gate-delay savings of a 16-bit two-level CLA vs ripple?
~5 vs ~32 gate delays — about 6× faster.
Why doesn't software run the same speed-up?
Hardware evaluates all gates in parallel. Software is sequential, so cleverness in algebra doesn't translate.
Verilog (6)
What's the difference between wire and reg?
A wire is a combinational signal driven by continuous assignments. A reg holds a value across a simulation step and can be assigned only inside always or initial blocks.
Difference between blocking (=) and nonblocking (<=) assignment?
Blocking assigns immediately and blocks the next statement. Nonblocking evaluates all RHSs first, then assigns all LHSs simultaneously — needed for sequential logic so updates feel atomic on the clock edge.
How do you write a rising-edge D flip-flop in Verilog?
always @(posedge clock) Q <= D; with Q declared as a reg.
What three rules keep an always block from synthesizing into a latch?
(1) Place all combinational logic in a continuous assign or always block. (2) Sensitivity list lists all inputs. (3) Every path through the always block assigns the same set of bits.
What does {32{2'b01}} evaluate to?
A 64-bit value 0101...01 (the 2-bit pattern 01 repeated 32 times).
Behavioral vs structural Verilog?
Behavioral describes what the logic does (synthesizes into something). Structural describes the circuit hierarchy explicitly (gates, modules, wires).
Clocking, latches, flip-flops (7)
What does "edge-triggered" mean?
All state changes occur on a clock edge (rising or falling), not while the clock is at a level.
Why is edge-triggered design preferred?
Simpler to reason about. Allows reading and writing the same state element in one cycle without race because the write latches at the edge while the read sees the old value.
D latch vs D flip-flop?
A D latch is transparent — Q tracks D while the clock is asserted. A D flip-flop changes only on a clock edge.
How is a falling-edge D-FF built from D latches?
Master-slave: a D latch open while clock is high feeds a second D latch open while clock is low. The output changes only when the clock falls.
Define setup and hold time.
Setup = minimum time the input must be valid before the clock edge. Hold = minimum time it must remain valid after the clock edge.
What is the formula for the minimum clock period?
T ≥ t_prop(FF) + t_comb_max + t_setup(FF), plus skew margin if needed.
If the same register file slot is read and written in one cycle, what value do you read?
The old value — the write doesn't take effect until the clock edge. To read the new value the same cycle, additional forwarding logic is needed.
SRAM, DRAM, ECC (8)
Why does SRAM use 4–6 transistors per bit while DRAM uses 1?
SRAM stores the bit on cross-coupled inverters (need both an inverter pair and access transistors). DRAM stores it as charge on a capacitor accessed by one pass transistor.
Why must DRAM be refreshed?
Because the capacitor leaks charge over a few milliseconds. The cell value would be lost without periodic read-then-rewrite.
What's the role of RAS and CAS?
Address pins are reused: RAS (Row Access Strobe) tells the DRAM the bits on the address pins are a row address; CAS does the same for column. Two-step decode lets one set of pins serve both axes.
Why does refresh consume so few cycles (1–2%)?
Because the row decode latches an entire row at once, refresh of all bits in one row happens in a single read-write cycle — far fewer cycles than touching every individual cell.
Why are tri-state buffers used in SRAM cell arrays?
Many cells share one bit line; only the selected cell drives it. Tri-state's high-impedance state lets all unselected cells "disconnect."
What does DDR mean?
Double Data Rate — transfers data on both rising and falling clock edges, doubling effective bandwidth.
What can a 1-bit parity code do and not do?
Detect any single-bit error (and any odd number). Cannot correct any error. Cannot reliably detect 2-bit errors (parity matches again).
Cost of SEC-DED for a 64-bit word?
7 extra bits per 64 data bits.
FSMs (5)
What functions describe an FSM?
A next-state function (state, inputs → next state) and an output function (state for Moore, or state + inputs for Mealy).
Difference between a Moore and a Mealy machine?
Moore outputs depend only on state. Mealy outputs depend on state and inputs. Mealy may need fewer states; Moore is simpler/faster.
What is "state assignment" and why does it matter?
Choosing the binary encoding of states. Smart assignment makes logic equations contain redundancy and minimize.
For the 2-state traffic light, when does it switch?
Only when a car is waiting in the opposite direction. Otherwise it stays in the current state.
What's the typical hardware structure for an FSM?
A state register holding the current state plus combinational logic (often a PLA or ROM) implementing the next-state and output functions.
Timing, skew, metastability (5)
What is clock skew?
The difference in absolute time at which two state elements see the same clock edge, caused by routing delay differences.
How does clock skew cause a race?
If skew exceeds the propagation delay through a flip-flop, the data from FF1 can reach FF2's input before FF2's clock edge arrives — FF2 captures the new value instead of the intended old one.
What is metastability?
A flip-flop's output settling in the indeterminate region between high and low for an unbounded time, after sampling an input that violated setup/hold.
What is synchronizer failure?
Different downstream gates reading the metastable output and disagreeing — some see 1, some see 0 — diverging system state.
How does a 2-FF synchronizer mitigate metastability?
FF1 may go metastable, but its output has one full clock period to resolve before FF2 samples it — making the probability of a metastable output reaching downstream logic vanishingly small (years/decades MTBF).
FPDs (4)
SPLD vs FPGA?
SPLD = single PLA or PAL (combinational only, simple). FPGA = array of LUT-based cells with flip-flops, configurable interconnect, supports complex sequential designs.
What's a LUT?
Lookup Table — a small RAM whose contents are the truth table of the function the cell implements.
Difference between PLA and PAL?
PLA has both AND and OR planes programmable. PAL has a programmable AND plane and a fixed OR plane.
What two configuration mechanisms are common in FPDs?
Antifuse (one-time, permanent) and SRAM-based (volatile, reconfigurable on power-up).

Common Exam Traps

  • "A latch and a flip-flop are the same thing." No. A latch is level-sensitive (transparent while clock is asserted). A flip-flop changes only on a clock edge.
  • "A 1-bit parity code can detect any 2-bit error." No. 2 errors flip parity twice, restoring the match. Parity detects only odd numbers of errors (in practice, single-bit).
  • "SRAM is faster than DRAM because it has more transistors per bit." Misleading. SRAM is faster because it stores state in inverters (no precharge, no sense amp, no refresh). Transistor count is a consequence, not the cause.
  • "DRAM is too slow to use because of refresh." No. Refresh consumes only 1–2% of cycles because whole rows refresh in one cycle. The other 98% are available for reads/writes.
  • "Carry lookahead computes carries serially, just faster gates." No. CLA computes all the carries in parallel using generate/propagate signals — that's where the speedup comes from.
  • "You can't read and write the same register in one clock cycle." You can. Edge-triggered registers permit it: the read sees the value stable before the edge, the write happens at the edge.
  • "Mealy machines are always smaller than Moore machines." Sometimes. Mealy can use fewer states, but it isn't guaranteed. The two are functionally equivalent.
  • "In Verilog, '<=' is the comparison operator." Not in this context. Inside an always block, <= is the nonblocking-assignment operator. (It IS still ≤ comparison in expressions.)
  • "Don't-cares are bugs in the spec." No. They are an intentional way to give the logic minimizer freedom to merge truth-table rows and reduce gate count.
  • "Two-FF synchronizers eliminate metastability." Reduce, not eliminate. Probability shrinks exponentially over time but never reaches 0. Designers tolerate a vanishingly small failure rate (years between failures).
  • "NOR can be built from AND and OR but not the reverse." Backwards. NOR is universal — AND, OR, NOT can all be built from NOR alone. Same for NAND.
  • "PLAs and ROMs implement different kinds of functions." No. Both implement combinational logic. The difference is encoding: ROM is fully decoded (one entry per input combination), PLA is sparse (one minterm per active product term).
  • "Edge-triggered means the state changes whenever the clock is high." No. That's level-sensitive. Edge-triggered means changes happen only at the rising or falling edge.
  • "A bus is just a wire." No. A bus is a collection of data lines treated as a single logical signal — and may be a shared resource with multiple drivers (using tri-state).
  • "FPGAs are mostly logic." No. ~90% of FPGA die area is interconnect; logic and memory blocks are only ~10%.
