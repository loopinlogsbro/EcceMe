3.1 Introduction vocabulary

To control hardware you must speak its language. The "words" are instructions; the "vocabulary" is the instruction set. The textbook teaches a subset of ARMv8 called LEGv8.

Instruction setVocabulary of commands a given architecture understands.
Stored-program conceptInstructions and data are both stored in memory as numbers, so programs can be loaded, copied, and changed like any other data.
ARMv8 vs LEGv8ARMv8 = full ARM 64-bit ISA. LEGv8 = teaching subset.

Likely exam phrasing: "Programs and data can be stored in memory as numbers" → True. "Instruction sets of different computers are quite similar" → True.

3.2 Operations of the computer hardware

Every LEGv8 arithmetic instruction has exactly three operands (one destination, two sources). This is rigid by design: simplicity favors regularity.

ADD a, b, c    // a = b + c
SUB a, b, c    // a = b - c

To sum four variables you need three ADDs (each ADD adds only two values):

ADD a, b, c    // a = b + c
ADD a, a, d    // a = b + c + d
ADD a, a, e    // a = b + c + d + e

Three design principles introduced in this chapter:

  • Simplicity favors regularity — fixed instruction format, fixed 3-operand arithmetic.
  • Smaller is faster — why LEGv8 has 32 registers (more registers ⇒ slower clock).
  • Good design demands good compromises — different instruction formats coexist (R/I/D/B/CB/IM) so opcodes stay regular while leaving room for immediates.

3.3 Operands of the computer hardware core

Word32 bits.
Doubleword64 bits — the size of a LEGv8 register.
RegisterFast hardware storage. LEGv8 has 32 registers, each 64 bits wide, named X0…X30 plus XZR.
MemoryLarge 1-D array of bytes. Byte-addressed: addresses of sequential doublewords differ by 8.
AddressIndex into memory; selects a byte.
XZRZero register (X31) — always reads as 0; cannot be changed.

Load / Store (data transfer)

LDUR  X9,  [X22, #8]     // X9 = Mem[X22 + 8]    (load doubleword)
STUR  X9,  [X22, #8]     // Mem[X22 + 8] = X9    (store doubleword)
LDURB X9,  [X0, #0]      // load 1 byte, zero-extend
STURB X9,  [X1, #0]      // store low byte
LDURH X9,  [X0, #0]      // load 16-bit halfword (sign-extends)
STURH X9,  [X1, #0]      // store halfword

Address = base register + immediate offset. Offset in [Xn, #imm] is 9 bits in D-format → ±256 byte range from base.

Compiling g = h + A[8];

// X20=g, X21=h, X22=base of A
LDUR X9,  [X22, #64]   // X9 = A[8]   (offset = 8 * 8 bytes = 64)
ADD  X20, X21, X9      // g = h + A[8]

Why 64? Doublewords are 8 bytes, so element index i uses byte offset i*8.

3.4 Signed and unsigned numbers core

LSBLeast-significant bit — rightmost (bit 0).
MSBMost-significant bit — leftmost (bit 63 in a doubleword).
Sign bitIn two's complement, the MSB. 0 = positive, 1 = negative.

Ranges

WidthUnsignedSigned (two's complement)
4-bit0 … 15−8 … 7
8-bit0 … 255−128 … 127
32-bit0 … 2³² − 1 ≈ 4.29 B−2³¹ … 2³¹ − 1
64-bit0 … 2⁶⁴ − 1−2⁶³ … 2⁶³ − 1
Unsigned N-bit max: 2^N − 1 Signed N-bit range: −2^(N−1) to 2^(N−1) − 1

Two's complement negation (memorize)

  1. Flip every bit (0↔1).
  2. Add 1.
+5  in 8-bit   = 0000 0101
flip           = 1111 1010
+1             = 1111 1011  =  -5

Sign extension

To extend an N-bit two's-complement value to more bits, copy the sign bit into the new high bits.

-2 in 32-bit:   1111 1111 ... 1111 1110
-2 in 64-bit:   1111 1111 ... 1111 1111 1111 ... 1111 1110

Overflow

For two's complement: overflow occurs when the operands have the same sign but the result has the opposite sign (i.e. the carry into the MSB ≠ the carry out of the MSB).

  • pos + pos = neg → overflow
  • neg + neg = pos → overflow
  • pos + neg → never overflows

Signed vs unsigned loads

  • LDURSB / LDURSW — sign-extend (treat byte/word as signed).
  • LDURB / LDURH — zero-extend (treat as unsigned).
  • Memory addresses are unsigned — there are no negative addresses.

3.5 Representing instructions in the computer core

All LEGv8 instructions are exactly 32 bits long. There are six formats; you mainly need R, I, D (and B, CB for branches).

R-format (register arithmetic / logical / shift)

| opcode | Rm  | shamt | Rn  | Rd  |
|  11    |  5  |   6   |  5  |  5  |   = 32 bits

Examples: ADD, SUB, AND, ORR, EOR, LSL, LSR, BR. shamt = shift amount (0 unless it's a shift).

I-format (immediate)

| opcode | immediate | Rn  | Rd  |
|  10    |    12     |  5  |  5  |   = 32 bits

Examples: ADDI, SUBI, ANDI, ORRI, EORI. Immediate is 12 bits (zero-extended in LEGv8).

D-format (data transfer / load & store)

| opcode | address | op2 | Rn  | Rt  |
|  11    |    9    |  2  |  5  |  5  |   = 32 bits

Examples: LDUR, STUR, LDURB, STURB, LDURH, STURH. Address = 9-bit signed byte offset. Rt is destination on load, source on store.

The "Big Picture"

  1. Instructions are represented as numbers.
  2. Programs are stored in memory and read/written like data.

This is the stored-program concept — it's why one machine can run any program (compiler, editor, game) just by loading different memory.

Hex shortcut

4 binary bits = 1 hex digit. C/Java hex prefix is 0x.

1010 1011 0000 0011  =  0xAB03

3.6 Logical operations

OperationC/JavaLEGv8Use
Shift left<<LSLMultiply by 2ⁿ; shamt in R-format. LSL Xd, Xn, #n
Shift right>>>LSRLogical right shift (fills with 0). Divide unsigned by 2ⁿ.
Bitwise AND&AND, ANDIMask — clear specific bits.

There is no NOT instruction in LEGv8. To invert: EOR Xd, Xn, X-with-all-1s (or use EORI with a constant of all-1s).

// Multiply X10 by 8
LSL X10, X10, #3      // 2^3 = 8

// Force the low bit of X10 to 1
ORRI X10, X10, #1

// Mask to keep only low byte
ANDI X10, X10, #0xFF

3.7 Instructions for making decisions core

CBZ Xn, labelCompare-and-branch if zero. Branches if Xn == 0.
CBNZ Xn, labelCompare-and-branch if not zero. Branches if Xn != 0.
B labelUnconditional branch (uses B-format, ±128 MB range).
B.cond labelBranch on flags after a "set flags" instruction (ADDS/SUBS/etc.).
BR XnUnconditional branch to address held in register Xn.

Condition codes (NZCV flags)

  • N — result Negative (MSB = 1)
  • Z — result Zero
  • V — signed oVerflow
  • C — Carry / borrow out of MSB

Flags are only set by the "S" variants: ADDS, ADDIS, ANDS, ANDIS, SUBS, SUBIS. Add an S to set flags. Plain ADD does not change flags.

Plus four flag-bit branches: B.MI (N=1), B.PL (N=0), B.VS (V=1), B.VC (V=0).

If/else pattern

// if (i == j) f = g + h; else f = g - h;
SUB  X9,  X22, X23     // X9 = i - j
CBNZ X9,  Else         // if i != j (X9 != 0) jump to Else
ADD  X19, X20, X21     // f = g + h
B    Exit
Else: SUB X19, X20, X21  // f = g - h
Exit:

While loop pattern

// while (save[i] == k) i++;
Loop: LSL  X10, X22, #3       // i * 8
      ADD  X10, X10, X25      // &save[i]
      LDUR X9,  [X10, #0]     // save[i]
      SUB  X11, X9,  X24      // save[i] - k
      CBNZ X11, Exit          // exit if not equal
      ADDI X22, X22, #1       // i++
      B    Loop
Exit:

Bounds check shortcut

Treating signed values as unsigned makes a single unsigned ≥ check verify both x < length and x ≥ 0 (because negatives become huge unsigned values).

// Check 0 ≤ X20 < X11, branch to error if out of range
SUBS XZR, X20, X11
B.HS IndexOutOfBounds

Basic block = straight-line code with no branches in the middle (label only at start, branch only at end). Compilers split programs into basic blocks early.

3.8 Supporting procedures core

Six steps of a procedure call:

  1. Caller puts parameters where the callee can find them.
  2. Transfer control to the procedure (BL).
  3. Acquire storage (push on stack if needed).
  4. Perform the work.
  5. Place return value where caller can read it.
  6. Return control (BR LR).

Branch-and-link / Return

BL   ProcedureAddress    // LR (X30) = PC + 4; PC = ProcedureAddress
BR   LR                  // PC = LR  (return)

The stack

  • Stack grows downward (toward lower addresses).
  • Push: SUBI SP, SP, #16; STUR Xn, [SP, #0]
  • Pop: LDUR Xn, [SP, #0]; ADDI SP, SP, #16
  • An activation record / procedure frame contains saved regs + local variables for one call.
  • FP (X29) points to the first doubleword of the frame; SP may move during the procedure, FP stays put.
  • ARMv8 wants the stack 16-byte (quadword) aligned.

Memory layout (Linux on LEGv8)

high addr  ┌──────────────┐
           │   Stack      │  grows down  (SP starts ~0x0000_007F_FFFF_FFFC)
           │      ↓       │
           │              │
           │      ↑       │
           │   Heap       │  malloc/new, grows up
           │              │
           │ Static data  │  globals, constants (~0x0000_0000_1000_0000)
           │ Text segment │  machine code   (starts at 0x0000_0000_0040_0000)
low addr   │ Reserved     │
           └──────────────┘

Leaf vs nested procedure

  • Leaf — calls nothing. May not need to save LR.
  • Non-leaf — must save LR (X30) on the stack before calling another procedure, then restore before returning.
  • Recursive procedures are non-leaf (they call themselves).

Static vs automatic vs heap

  • Static — globals; lives in static data segment.
  • Automatic — local variables; on the stack, freed when procedure returns.
  • Heap — malloc in C / new in Java; explicit (or GC) free.

3.10 LEGv8 addressing — wide immediates & addresses core

Wide constants: MOVZ / MOVK

The 12-bit immediate in I-format isn't enough for big constants. LEGv8 has two move-wide instructions that load 16 bits at a time into one of four 16-bit "quadrants" (LSL 0/16/32/48):

MOVZ X9, 255, LSL 16     // X9 = 0x00000000_00FF0000  (zero rest)
MOVK X9, 8,   LSL 0      // X9 = 0x00000000_00FF0008  (keep rest)

To build a 64-bit constant: one MOVZ + up to three MOVKs.

Branch addressing

  • B / BL (B-format): 26-bit field, treated as word offset → reaches ±128 MB from PC.
  • CBZ / CBNZ / B.cond (CB-format): 19-bit field, word offset → ±1 MB from PC.
  • All conditional branches use PC-relative addressing: target = PC + (offset × 4).
  • For longer reach, the assembler rewrites a far conditional branch into CBNZ Xn, skip; B faraway; skip:.

The four LEGv8 addressing modes

3. Base / displacementMemory at Reg + constantLDUR, STUR, LDURB…
4. PC-relativeMemory at PC + (constant × 4)B, BL, CBZ, CBNZ, B.cond

3.11 Parallelism & instructions: synchronization

Threads cooperate by reading values others write. Without ordering, you get a data race: two accesses from different threads to the same location, at least one a write, occurring back-to-back with unpredictable result.

LEGv8 provides an atomic load-store pair for building locks/mutexes:

  • LDXR Xt, [Xn] — load exclusive register (mark address).
  • STXR Xs, Xt, [Xn] — store exclusive: if no other write happened to that address since the LDXR, store succeeds and Xs = 0; else Xs = 1 and the store is discarded.

Loop until STXR returns 0 to implement test-and-set / compare-and-swap on top.

Self-test (likely exam-style Q&A)

Number representation
What is the largest unsigned value representable in N bits?
2^N − 1.
Two's complement range for 32 bits?
−2,147,483,648 to 2,147,483,647 (i.e. −2³¹ to 2³¹ − 1).
How do you negate a value in two's complement?
Flip every bit, then add 1.
How is sign extension done?
Replicate the MSB (sign bit) into every new high-order bit.
A two's-complement add overflows when…?
The two operands have the same sign but the result has the opposite sign (carry-in to MSB ≠ carry-out).
In two's complement, how is 0 represented?
All zeros — there is only one zero.
Convert binary 1011 to decimal.
1·8 + 0·4 + 1·2 + 1·1 = 11.
Hex 0xD6 to binary?
1101 0110.
Instructions & operands
How many operands does every LEGv8 arithmetic instruction take?
Exactly three (one destination, two sources).
How many registers does LEGv8 have, and how wide is each?
32 registers, each 64 bits.
Why limit to 32 registers?
"Smaller is faster" (signal travel time) plus more registers cost more bits in every instruction's register fields.
Difference between word and doubleword?
Word = 32 bits; doubleword = 64 bits (size of a LEGv8 register).
What does LDUR X9, [X22, #8] do?
Loads the doubleword at memory address (X22 + 8) into X9.
For element A[i] of a doubleword array, what byte offset goes in the LDUR?
i × 8.
Which register always reads as zero?
XZR (X31).
Instruction formats
Length of every LEGv8 instruction?
32 bits.
Format of ADD?
R-format. Fields: opcode | Rm | shamt | Rn | Rd.
Format of ADDI?
I-format (immediate). 12-bit immediate field.
Format of LDUR / STUR?
D-format (data transfer). 9-bit address field.
In LDUR, which field names the destination register?
Rt (data transfers use Rt, not Rd, because in STUR the field is the source).
Why are there multiple instruction formats?
Compromise: keep all instructions 32 bits, but make room for immediates/addresses by using different field layouts.
Branches & decisions
CBZ vs CBNZ?
CBZ branches if the register == 0; CBNZ branches if != 0.
Which addressing mode do conditional branches use?
PC-relative.
What sets the NZCV flags?
Only the "S" variants of arithmetic/logical instructions: ADDS, ADDIS, ANDS, ANDIS, SUBS, SUBIS.
How do MIPS conditional branches differ from LEGv8?
MIPS compares two registers directly in the branch instruction; LEGv8 uses condition-code flags set by a prior S instruction.
What's a basic block?
A straight-line sequence with no branch in the middle (label only at start, branch only at end).
Branch range of B / BL? Of CBZ / CBNZ?
B/BL: ±128 MB. CB-format: ±1 MB.
Procedures & stack
Which register holds the return address after BL?
LR (X30) — BL stores PC + 4 there.
How do you return from a procedure?
BR LR.
Which registers must a callee save if it modifies them?
X19–X27 (saved), X28 SP, X29 FP, X30 LR.
Argument registers?
X0–X7. Same regs return values.
Direction the stack grows?
Downward (toward lower addresses).
What's an activation record?
The frame on the stack for one call — saved regs + locals + params.
Leaf vs non-leaf procedure?
Leaf calls no other procedure (often need not save LR). Non-leaf must save LR before its inner BL.
What lives on the heap vs stack?
Heap = dynamic (malloc / new). Stack = automatic locals, frames, saved regs.
Logical / shifts / ASCII
How do you implement NOT in LEGv8?
EOR with a register of all 1s (no NOT instruction).
LSL by n is equivalent to multiplying by what?
2^n.
Which logical op forces specific bits to 0?
AND with a mask.
Which logical op forces specific bits to 1?
OR with a mask.
ASCII codes for 'A', 'a', '0'?
65, 97, 48 (uppercase & lowercase differ by 32).
How does C terminate a string?
A null byte (value 0). Java stores an explicit length instead.
Translation & loading
Order of the four translation tools?
Compiler → Assembler → Linker → Loader.
What does the linker do?
Combines object files + libraries, resolves all symbol references, relocates absolute addresses, produces the executable.
What is a pseudoinstruction?
An assembler convenience that maps to one or more real machine instructions (e.g. MOV → ORR with XZR).
Why use DLLs?
Smaller executables, library updates without relinking, on-demand loading. Cost: small first-call overhead via lazy linkage.
How is Java executed?
Compiled to bytecode → run on the JVM (interpreter) → hot methods JIT-compiled to native at runtime.
Standard UNIX suffixes for source / asm / object / shared lib?
.c / .s / .o / .so.

Common exam traps

  • "ADD sets the flags." False. Only the S-suffix versions (ADDS, SUBS, ANDS, ADDIS, SUBIS, ANDIS) update NZCV.
  • "There's a NOT instruction." No. Use EOR with all-ones.
  • "Doubleword addresses go up by 1 each." No. Memory is byte-addressed; sequential doublewords differ by 8.
  • "Two's complement has +0 and −0." No. Only one zero; that's the whole reason it beats sign-and-magnitude.
  • "CBZ branches if the register is non-zero." No — CBZ branches if zero. CBNZ branches if non-zero.
  • "Conditional branches reach the entire program." No. ±1 MB only (CB-format). Far branches are split into a flipped condition + an unconditional B.
  • "Java compiles directly to native code." Not initially. Compiles to bytecode; JVM interprets; JIT compiles hot methods later.
  • "The linker is the same as the loader." No. Linker resolves symbols and produces the executable; loader (part of OS) puts the executable into memory and starts it.
  • "X0 is preserved across a call." No. X0–X7 are argument/return regs and not preserved. Saved set is X19–X27 (plus SP/FP/LR).
  • "Adding two negatives can overflow into a positive." Yes — that's textbook signed overflow.
