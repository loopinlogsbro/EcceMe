4.5 LAB: Max of 3 — branch core

Find the maximum of three values held in X19, X20, X21 and store the result in X24.

X19X20X21X24 (max)
5989

Approach: compare-and-branch using SUBS to set flags, then B.GT/B.LT to skip the loser. The SUBS family sets condition flags; plain SUB does not.

// Sketch
ADD  X24, XZR, X19      // assume X19 is max
SUBS XZR, X24, X20      // compare with X20
B.GE skip1
ADD  X24, XZR, X20
skip1:
SUBS XZR, X24, X21      // compare with X21
B.GE done
ADD  X24, XZR, X21
done:

XZR is the zero register — reads as 0, ignores writes. Useful as a throwaway destination for "compare-only" SUBS.

4.6 LAB: Array of Fibonacci sequence — loop core

Populate an array with Fibonacci numbers. Sequence starts 0, 1, 1, 2, 3, 5, 8, 13, …; each later term is the sum of the previous two. Array size is always ≥ 1.

X19Size of the array (number of elements to write).

Loop pattern: keep "prev" and "curr" in two registers, write current to memory, then next = prev + curr, prev ← curr, curr ← next. Use CBZ/SUBS+B.GT to control loop count, and ADDI with #8 to advance the pointer.

4.7 LAB: Multiplications and divisions using shift operations core

Bitwise shifts as fast multiply/divide for positive integers.

Shift left by i bits ↔ multiply by 2^i Shift right by i bits ↔ divide by 2^i (integer division — fractional part dropped)

Given N in X19, store N * 8 in X20 and N / 16 in X21.

X19X20 = N×8X21 = N/16
645124
LSL X20, X19, #3   // *8  = shift left by 3
LSR X21, X19, #4   // /16 = shift right by 4

Multiplier 8 = 2^3, divisor 16 = 2^4. Counting the exponent is the trick the exam wants.

int Sum(int a, int b){ return a + b; }
int main(){

    return 0;   // do not implement
}
RegisterVariable
X19x
X20y
X21w
X22z

Calling convention (per the lab hint)

  • X0, X1 hold arguments.
  • X2 holds the return value.
  • X30 is the link register; BL writes return PC into X30, BR X30 returns.
// Provided callees (do not modify)
Sum: ADD X2, X0, X1
     BR  X30
Dif: SUB X2, X0, X1
     BR  X30

Main:
ADD X0, XZR, X19      // arg0 = x

BL  Sum               // X2 = x + y

ADD X0, XZR, X20      // arg0 = y

4.10 LAB: Nested procedures core

Implement procedure Sum which itself calls Dif twice. Because Sum calls another procedure, X30 must be saved/restored across the inner calls (the stack pointer is initialized via ADDI X28, XZR, #1000 in the provided template).

// Provided C
int Dif(int a, int b) { return b - a; }
int Sum(int m, int n) {
    int p = Dif(n+1, m-1);
    int q = Dif(m+1, n-1);
    return p + q;
}
int main() {
    int x, y;
    z = x + y + Sum(x, y);
    return 0;
}
RegisterVariable
X19x
X20y
X21z (result)

Sample I/O

X19 (x)X20 (y)X21 (z)
51011

Provided template skeleton

ADDI X28, XZR, #1000  // initialize stack pointer (X28 = SP region)
// Procedure Main (do not modify)
Main:
ADD X0, XZR, X19
ADD X1, XZR, X20
BL  Sum
ADD X21, X20, X19
ADD X21, X21, X2
B   End

In Sum you must save X30 (and any callee-saved values you reuse) on the stack before the first BL Dif, and restore them before BR X30 — otherwise the inner BL overwrites your return address.

Quick-reference cheat sheet last 10 minutes

Memory math

  • Doubleword stride = 8 bytes (used in 4.4, 4.6).
  • 4.8 uses an unusual stride 16 (offsets 0, 16, 32).
  • LDUR Xd, [Xn, #imm] → Xd = Mem[Xn+imm].
  • STUR Xs, [Xn, #imm] → Mem[Xn+imm] = Xs.

Shifts as mul/div

  • LSL Xd, Xn, #i = multiply by 2^i.
  • LSR Xd, Xn, #i = divide by 2^i (integer, drops fraction).
  • ×8 → LSL #3; /16 → LSR #4.

Branching

  • SUBS sets flags; SUB does not.
  • Use SUBS XZR, Xa, Xb for compare-only.
  • CBZ Xn, label / CBNZ Xn, label — branch on register zero / non-zero.

Procedure calling convention

  • X0, X1 = first two arguments.
  • X2 = return value (per the lab hint).
  • X30 = link register; BL label sets X30 = PC+4 then jumps.
  • BR X30 returns to caller.
  • X28 initialized as the stack pointer region in 4.10 (ADDI X28, XZR, #1000).
  • XZR = always-zero register.

Instruction groups (4.1)

  • Arith: ADD/ADDS/ADDI/ADDIS, MUL, SDIV, SUB/SUBS/SUBI/SUBIS.
  • Data: LDUR(B/H/SW), STUR(B/H/W), MOVK, MOVZ.
  • Logical: AND/ANDI, EOR/EORI, ORR/ORRI, LSL, LSR.
  • Cond branch: B.cond, CBZ, CBNZ.
  • Jump: B, BL, BR.

Simulator workflow

  • + under Registers / Memory to add and seed values.
  • STEP for single-step, RUN for full speed.
  • Do not initialize regs/mem in code — auto-grader handles that.

Self-test (likely exam-style Q&A)

Instruction set & simulator basics (4.1)
Which arithmetic instructions are supported by the zyLab platform?
ADD, ADDS, ADDI, ADDIS, MUL, SDIV, SUB, SUBS, SUBI, SUBIS.
Which data-transfer instructions are supported?
LDURB, LDUR, LDURH, LDURSW, MOVK, MOVZ, STURB, STUR, STURH, STURW.
Which logical instructions are supported?
AND, ANDI, EOR, EORI, ORR, ORRI, LSL, LSR.
Which unconditional jump instructions are supported?
B, BL, BR.
Should you initialize registers and memory inside your assembly code?
No. The auto-grader has its own initialization mechanism; initializing in code breaks grading.
What does LDUR X9, [X19, #0] do?
Loads the doubleword at memory address X19+0 into register X9.
In 4.1, where is variable A stored, and where must B be stored?
A at memory address 4000, B at 4008. X19=4000, X20=4008.
How do you add a register or memory cell in the simulator UI?
Click the + button under Registers or Memory and enter the register name or address in the popup.
Arithmetic labs (4.2, 4.3)
In 4.2, which register holds the result Z?
X23. Inputs A,B,C,D are in X19, X20, X21, X22.
In 4.2, which register is the only allowed temporary?
X9.
In 4.3, which registers map to A, B, C, D, Z?
X9=A, X10=B, X11=C, X12=D, X19=Z.
Which mnemonic performs signed integer multiplication?
MUL (signed division is SDIV).
Given X9=2, X10=4, X11=6, X12=3, the lab expects X19 = 18 — what does that imply about the expression?
It must combine those four inputs to evaluate to 18 (e.g. it includes a multiplication; the exact form is given in the source figure).
Loads, stores, arrays (4.4, 4.6, 4.8)
Why does the array of squares in 4.4 step memory addresses by 8?
Because each element is a 64-bit doubleword (8 bytes).
If X19=4000, what's the address of the third element of the source array in 4.4?
4000 + 16 = 4016.
In 4.4 with X20=5000 and source [1,2,3,4], what does memory at 5024 hold after the program runs?
16 (4 squared).
What is the first Fibonacci value written in 4.6, and to which address if X20=5000?
0, written to address 5000.
For X19=5, X20=5000 in 4.6, what does memory at 5032 contain?
3 (sequence: 0,1,1,2,3).
In 4.8, which addresses hold Length, Width, Height?
4000, 4016, 4032.
In 4.8, where is the result stored?
Memory address 4400.
Why are the 4.8 dimension offsets 0/16/32 instead of 0/8/16?
The lab specifies that exact memory layout; the LDUR offsets must match those addresses literally.
Branching and loops (4.5, 4.6)
Where does 4.5 store the maximum of three values?
X24 (inputs are X19, X20, X21).
Why use SUBS instead of SUB when comparing?
SUBS sets the condition flags so that B.GT/B.LT/B.GE etc. work; SUB does not update flags.
What's a clean idiom for "compare only, throw away the difference"?
SUBS XZR, Xa, Xb — XZR discards the result while flags still update.
Which two instructions branch based purely on whether a register is zero?
CBZ (branch if zero) and CBNZ (branch if non-zero).
In 4.6, which register holds the loop count and which holds the array address?
X19 holds the size, X20 holds the base address.
Shifts as mul/div (4.7)
Shifting left by i bits is equivalent to what arithmetic operation on a positive integer?
Multiplying by 2^i.
Shifting right by i bits is equivalent to what?
Integer division by 2^i (the fractional part is dropped).
How do you compute N×8 with a shift?
LSL Xd, Xn, #3.
How do you compute N/16 with a shift?
LSR Xd, Xn, #4.
If X19=64, what end up in X20 and X21 after N×8 and N/16?
X20=512, X21=4.
What happens to digits after the decimal point during LSR?
They are discarded — LSR performs integer division.
Procedure calls (4.9, 4.10)
Per the lab hint, which registers carry procedure arguments and which carries the return value?
X0 and X1 are the arguments; X2 is the return value.
What does BL do?
Branch and link — jumps to the label and stores the return address in X30 (the link register).
How does a procedure return?
BR X30 — branches to the address held in the link register.
In 4.9, in what order do you set up arguments before BL Dif for z = Dif(y, x)?
X0 ← y (X20), X1 ← x (X19), then BL Dif. Result lands in X2.
Why must Sum in 4.10 save X30 before calling Dif?
Because BL Dif overwrites X30. Without saving it, Sum loses its own return address and BR X30 will jump back to the wrong place.
What does the line ADDI X28, XZR, #1000 in the 4.10 template do?
Initializes X28 (used as the stack pointer) to 1000, giving Sum a place to push X30 and any other saved registers.
For 4.10 with X19=5, X20=10, what is the expected X21?
11.

Common exam traps

  • "Use SUB to compare values, then branch with B.GT." No. SUB does not set flags. Use SUBS (or SUBIS) to update NZCV before any conditional branch.
  • "Initialize X19 = 4000 inside the assembly so the auto-grader sees it." No. The auto-grader initializes registers and memory itself; setting them in code can cause all test cases to fail.
  • "Each array element is 4 bytes, so step the address by 4." No. The 4.4/4.6 labs explicitly use doubleword (8-byte) addresses; step by 8.
  • "4.8 uses the same 8-byte stride as 4.4." No. 4.8 places Length at 4000, Width at 4016, Height at 4032 — offsets 0, 16, 32.
  • "LSR rounds to the nearest integer." No. LSR truncates — the fractional part is dropped (integer division).
  • "To multiply by 8, use LSL #8." No. 8 = 2^3, so it's LSL #3. To divide by 16 = 2^4, it's LSR #4.
  • "BL only jumps; you must manually save the return address." No. BL automatically writes the return PC into X30. You only need to save X30 yourself when the called procedure makes another BL (as in 4.10's Sum).
  • "Return values come back in X0." No for these labs. Per the 4.9/4.10 hint, the convention used is X2 for return value, X0/X1 for arguments.
  • "In 4.5, store the max in X20." No. The destination is X24; X19/X20/X21 hold the inputs.
  • "You can use NOT to bitwise invert a register." No. NOT is not in the supported list; emulate with EOR against an all-ones value if needed.
  • "XZR can hold any value you write to it." No. XZR always reads as 0; writes to it are discarded — that's why SUBS XZR, Xa, Xb is the standard compare idiom.
