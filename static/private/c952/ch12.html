zyBook flags this whole chapter as Optional. It is COD Appendix D, a quick survey. The unifying theme: 10 RISC ISAs are remarkably similar — most "uniqueness" boils down to small differences in branch handling, immediates, and register conventions.

12.1 Introduction core

"RISC: any computer announced after 1985." The 10 architectures fall into two groups:

Two Groups

Desktop / Server RISCsEmbedded RISCs
  • Digital Alpha
  • HP PA-RISC
  • IBM/Motorola PowerPC
  • MIPS Inc. MIPS-64
  • Sun SPARC
  • ARM (ARM v4)
  • ARM Thumb (v1)
  • Hitachi SuperH (SH-3)
  • Mitsubishi M32R (v1)
  • MIPS-16 (v1)

Headline Differences key

  • Desktop/server RISCs: 32 GPRs, all instructions 32 bits, modern versions are flat 64-bit address.
  • Embedded RISCs: 8–16 GPRs, instructions 16 or 32 bits.
  • Thumb and MIPS-16 are not standalone ISAs — they are 16-bit modes of ARM/MIPS, switched into via a call. Procedures must be all 16-bit or all 32-bit (no mixing inside a procedure).

Survey order in the book: addressing modes & formats → MIPS core subset → multimedia extensions → DSP extensions → common extensions → unique features per ISA.

12.2 Addressing Modes & Instruction Formats core

Simplified addressing modes are a defining RISC trait. All desktop RISCs reserve one register that always reads as 0; this lets absolute and register-indirect be synthesized from displacement with a 0 base or 0 offset (PowerPC's r0 differs slightly — usable, can change).

Desktop vs Embedded Addressing

Desktop RISCsMostly base + displacement only. PA-RISC adds short offset variants. MIPS-64 adds indexed for FP load/store.
Embedded RISCsNo reserved zero register. ARM and SuperH offer richer modes, including shift-and-add forms; ARM can shift one register, add to another, and write back to one in a single mode.

PC-Relative Branches

  • Desktop RISCs (32-bit aligned instructions): branch offset is shifted left 2 bits before adding to PC → 4× the reach.
  • Embedded RISCs (16-bit aligned): offset is shifted left 1 bit → 2× the reach.
  • Procedure return / case / function-pointer: jump-register-indirect.

Instruction Formats

GroupNumber of formatsNotes
Desktop/server RISCs4 primary formatsR-R, R-immediate (data transfer), branch, jump/call. Alpha has a separate format for arithmetic immediates (8-bit immediate in bits 20:13).
Embedded RISCs6 formatsMore formats because mixing 16- and 32-bit instructions and 8/16 register sets demands extra encoding.

Constant Extension Tricks

  • Most desktops: sign-extend immediates to 64 bits (or 32). Exception: MIPS jump/call constants are not sign-extended — they replace the lower 28 bits of PC.
  • 16-bit-instruction ISAs have only 5–8 bit immediates, so most provide a 2-instruction sequence to build a long address (e.g., load upper, load lower).
  • ARM 8-bit immediate trick: the 8-bit value is rotated right by an even number (2–30) chosen by 4 bits in the instruction. Result: every power of two is encodable as a single immediate.

12.3 Instructions: The MIPS Core Subset core

~90% of executed instructions in any of these 10 ISAs are equivalent to the MIPS core. Categories:

  1. Data transfer
  2. Arithmetic / logical
  3. Control
  4. Floating point (mostly absent on embedded RISCs)
  5. Conventions (register usage, pseudoinstructions)

Compare-and-Branch Strategies key — the biggest divergence

ISAMechanism
SPARCFour condition codes (N, Z, C, V) in PSW — optional per instruction. Explicit compare = subtract with r0 destination. v9 added 64-bit CC set, register-test branches, FP CCs, branch-prediction bit.
PowerPCEight 4-bit CC registers (LT, GT, EQ, SO). Compare may target any of the 8. CR-logical instructions (CRAND, CROR, CRXOR, etc.) combine CCs.
MIPSNo integer CCs. BEQ/BNE compare two GPRs. SLT/SLTI/SLTU/SLTIU compute "less than" into a register. Special compare-with-zero branches (BGEZ, BGTZ, BLEZ, BLTZ). FP uses CCs (1 in MIPS I, 8 in MIPS IV).
AlphaCMPxx writes 1/0 into a GPR. FP compares write 2.0/0.0. Branches compare a register against 0 or test its LSB.
PA-RISCMany branch options including COMB (compare and branch). Heavy use of nullification.
ARMFour CCs, optionally set. CMP = subtract, CMN = add, TST = AND, TEQ = XOR. Every instruction is conditionally executable on CCs.
ThumbSame CCs as ARM, but setting them is not optional and there's no conditional execution of arbitrary instructions.
SuperHSingle T-bit set by compares. Branch BT if T=1, BF if T=0.

12.4 Multimedia (SIMD) Extensions for Desktop RISCs

By partitioning the carry chain in a 64-bit ALU, the same hardware can do parallel ops on packed bytes (8B), halfwords (4H), or words (2W). Origin: Intel i860 (1989). Sometimes called subword parallelism, vector, or SIMD; "SIMD" stuck because Intel marketed MMX that way.

Per-ISA Multimedia Support

ISAExtension nameNotes
HP PA-RISCMAX24H ops; first desktop RISC to add multimedia. Includes average instruction.
SPARCVIS4H or 2W. Includes constant-set instructions, memory alignment ops.
MIPSMDMX8B or 4H. Adds a 192-bit (3×64) accumulator partitioned to 8B×24 or 4H×48 bits. Vector/scalar variants. Claimed 2–4× speedup.
AlphaMAXMinimal: byte versions of compare, min, max, abs-diff. Software does the rest.
PowerPC(holdout)Was last to adopt multimedia.

Saturating Arithmetic key

On overflow, the result clamps to MAX or MIN instead of wrapping (mod). Common in DSP filtering. Helpful for audio/video where wrap-around would be a worse error than clipping.

Common across all four: only AND/OR/XOR — they don't need a partitioned ALU. Compilers can't easily target multimedia portably; these ops are intended for hand-written subroutine libraries.

12.5 DSP Extensions for Embedded RISCs

The defining DSP feature: integer multiply-accumulate (MAC). Multiplies are typically narrow (16 bits) and accumulators are wide (≥ 32, often 64). MAC efficiently implements digital filters.

  • Accumulators between 32 and 64 bits perform saturation on overflow (force max/min fixed-point).
  • Thumb and MIPS-16 are subsets — they have no DSP extensions; programmers use the 32-bit ARM/MIPS-64 modes for DSP work.

12.6 Common Extensions to the MIPS Core core

Instructions not in the MIPS core but found in two or more ISAs.

12.7 Instructions Unique to MIPS-64 core

  • Misaligned word transfers: LWL/LWR/SWL/SWR (and 64-bit forms) load/store a word that crosses an alignment boundary in just two instructions.
  • NOR = ~(Rs1 | Rs2).
  • Constant shift amount uses a 5-bit field in the R-format.
  • SYSCALL traps into the OS.
  • CTCi/CFCi move data between integer and control registers.
  • Jump/call NOT PC-relative — 26-bit address shifted left 2 and replaces low 28 bits of PC. Only matters near a 256 MB boundary.
  • TLB instructions: MIPS I handled TLB misses in software; instructions to manipulate TLB regs are coprocessor ops, vary across MIPS versions.
  • Reciprocal & reciprocal sqrt — fast, not IEEE-754 rounded.
  • BGEZAL / BLTZAL — conditional procedure-call (branch-and-link if ≥0 or <0). Gives PC-relative call.
  • Paired-single FP: two parallel 32-bit FP ops on a 64-bit register: ADD.PS, SUB.PS, MUL.PS, MADD.PS, MSUB.PS, etc.
  • FP can run in parallel with integer if early checks confirm no FP exception possible (otherwise serialized).

12.8 Instructions Unique to Alpha

Two original controversial choices: imprecise FP exceptions and no byte/halfword loads/stores (until 21164A).

  • TRAPB — stalls until prior arithmetic instructions are guaranteed exception-free. Used to enforce precise FP exceptions; one TRAPB per FP op slows code ~5×.
  • NaN, infinity, and denormal handling are done in software → require precise interrupts → require TRAPB.
  • Byte manipulation: EXTxx (extract field), INSxx (insert field), MSKxx (mask), ZAP (zero fields), CMPGE (compare bytes).
  • PAL code = privileged architecture library. Mode with all privileges, interrupts off, VM mapping off. Used for TLB management, atomic ops, OS primitives. Invoked via CALL_PAL.
  • No integer divide.
  • LDQ_U/STQ_U — 64-bit unaligned load/store using addresses with low 3 bits ignored; combine with extract instructions.
  • Single-precision FP held as double in registers; converted on load/store.
  • FP register F31 = 0 (hardwired).
  • VAX FP formats F and G are supported in addition to IEEE 754 S and T (D dropped — exponent too narrow).
  • Bit-count instructions: CTLZ, CTTZ, CTPOP (added in v3) — useful for crypto.

12.9 Instructions Unique to SPARC v9 core

Register Windows key

The defining SPARC feature. Multiple banks of registers; each procedure call allocates a new window. Banks act as a circular buffer (no fixed call depth limit; sweet spot 6–8 banks). Each window typically holds 8 globals + 8 locals + 8 incoming + 8 outgoing parameters = 16 unique regs/window. Implementations may have 40–520 physical registers (typically 128–136).

  • Window changes use separate SAVE/RESTORE instructions, not call/return.
  • SAVE: source from caller's window, dest in callee's window — also used to allocate stack frame.
  • RESTORE: inverse — dest in caller's window, deallocates frame; also used for return-value setup.
  • Doesn't necessarily slow the clock — cache access dominates register access in practice.

Other SPARC Features

  • Alternate-space load/store — distinguish memory accesses to I/O devices and control registers.
  • Fast traps: ≥ 4 trap levels so window over/underflow handlers can themselves be interrupted; RETRY, DONE, and RETURN instructions support multilevel trap return.
  • Tagged add/subtract (TADDcc, TSUBcc) — designed for LISP/Smalltalk. Low 2 bits = type tag; 00 = integer. Sets overflow if either operand is non-integer or result too large.
  • Overlapped int/FP execution: queue of pending FP instructions; RDPR empties it.
  • FSQRTS/FSQRTD/FSQRTQ — FP square root.
  • JMPL Rd — Rd specifies the return address register (Rd = r31 ≈ MIPS JALR; Rd = r0 ≈ MIPS JR).
  • LDSTUB — atomic byte load + store 0xFF (synchronization primitive, v8).
  • CASA/CASXA — atomic compare-and-swap on 32 / 64 bits (v9). Foundation for wait-free synchronization.
  • BPcc/BPr/FBPcc — branches with prediction-hint bit.
  • ILLTRAP — illegal instruction trap (used for aggregate returns in C).
  • POPC — population count.
  • Nonfaulting loads — speculative loads that won't fault if the address is bad.
  • Quad-precision FP — registers act as 8×128-bit.
  • XNOR — XOR with complement of operand 2.

12.10 Instructions Unique to PowerPC

Link & Count Registers key

  • Link register: dedicated register holds return address on call. No reg file read needed for return → faster jumps.
  • Count register: dedicated register for loop iteration counts. Branch-on-count auto-decrements. Branch hardware sees the value early in the cycle → easy prediction.
  • Both registers can also serve as branch target address sources: BCLR (branch via link register), BCCTR (via count register).

Other PowerPC Quirks

  • Register 0 is not hardwired to 0. As a base register it generates 0; as an index in base+index it works normally.
  • Load/store multiple — save/restore up to 32 regs in one instruction.
  • LSW/STSW — fixed/variable-length string transfers, arbitrary alignment.
  • Rotate-with-mask — extract or insert a bit field by rotating then masking. Two flavors.
  • Algebraic right shift — sets carry (CA) if operand is negative and 1-bits are shifted out → enables signed power-of-2 divide via SRAWI; ADDZE.
  • CBTLZ count leading zeros.
  • SUBFIC = immediate - RA → builds 1's/2's complement.
  • Logical shifted immediate — shift the 16-bit immediate left 16 bits before AND/OR/XOR.

12.11 Instructions Unique to PA-RISC 2.0

Has the most addressing modes, the most instruction formats, and many "two-instructions-in-one" hybrids of any desktop RISC.

Nullification key

Generalized to any arithmetic/logical instruction or branch. An ADD can compute, store, AND skip the next instruction if the sum is zero. Eliminates branches around single-instruction then bodies.

Lots of Conditional Branches

Despite nullification, PA-RISC has the largest set of conditional branches of any RISC. 16 conditions: =, <, ≤, odd, signed overflow, unsigned no-overflow, zero or no-overflow unsigned, never, plus complements. BB branches on a selected bit; BVB uses the shift-amount register.

Other PA-RISC Quirks

  • Synthesized multiply/divide — primitives shift one operand 1/2/3 bits and add (mul step). Divide step does nonrestoring divide. Avg multiply-by-constant ≈ 6 cycles, multiply-by-variable ≈ 24 cycles. PA-RISC has 10 such mul/div primitive instructions.
  • Decimal ops for COBOL: 4 bits/digit. 5 instructions provide adjust/condition-code support after a binary add.
  • Branch vectored — index reg shifted left 3, added to base, branch to result. For switch statements.
  • Extract / deposit — 12 instructions for bit-field manipulation.
  • ADDIL — add a left-adjusted 21-bit constant to a register, place result in r1. Followed by data transfer with 11-bit offset → encodes 32-bit constant addressing in 2 instructions.
  • 9 debug instructions for breakpoints on instruction or data addresses.
  • Load-and-clear — semaphore: read value, write 0.
  • Store bytes short — unaligned data move, leftmost or rightmost bytes.
  • Cache hints on loads/stores. Load with destination = r0 = software-controlled cache prefetch. PA-RISC 2.0 extended hints to stores (for block copy: don't load into cache).
  • Branch target stack (optional in 2.0) for indirect jump prediction (subroutine returns).
  • Multiply/add and multiply/subtract — two FP ops per instruction, plus fused multiply-add and fused multiply-negate-add.

12.12 Instructions Unique to ARM core

Conditional Execution of EVERY Instruction key

Each instruction starts with a 4-bit condition field. If the condition is false, the instruction acts as a NOP. Conditional branches are just conditionally-executing unconditional branches. Saves space and time by avoiding branches around single instructions.

12-Bit Immediate Trick

Low 8 bits zero-extended to 32, then rotated right by (high 4 bits × 2) bits. Lets every power of two fit in a single immediate.

Operand Shifting

The 2nd register operand of any arithmetic/logical instruction may be pre-shifted (LSL, LSR, ASR, ROR). Enables rotate-and-add, shift-right-and-test, etc., as one-cycle fused ops.

Other ARM Features

  • Block load/store (LDM/STM) — 16-bit register-mask in the instruction; loads or stores any subset of the 16 GPRs in a single instruction. Used for procedure entry/return and memory copy.
  • RSB (reverse subtract) and RSC (with carry) — first register subtracted from immediate or shifted register.
  • Long multiplies: SMULL/UMULL produce 64-bit signed/unsigned product into Hi/Lo regs.
  • No integer divide.
  • Conditional trap is free (every instruction including SWI is conditional).
  • Coprocessor interface: data transfer, GPR↔coproc moves, coproc ops. FP architecture (FPA10) was originally implemented as a coprocessor.
  • BX (branch and exchange) — switches between ARM and Thumb modes. Lower 31 bits of register → PC; MSB selects ARM (1) / Thumb (0).

12.13 Instructions Unique to Thumb

Thumb is a 16-bit-instruction subset of ARM, translated by hardware into ARM ops. Hot procedures use ARM (performance), cold procedures use Thumb (code density).

Restrictions vs ARM

  • No conditional execution on most instructions.
  • Easy access to only the first 8 registers; SP, LR, PC are used implicitly.
  • Two-operand format (instead of three) to save space.
  • Shifted-immediate and shifted-second-operand are gone — use separate shift instructions.
  • Simplified addressing modes; many more instruction formats due to 16-bit size.
  • Logical immediates dropped.
  • Condition codes are implicit by opcode: all ALU ops set them, no data transfers do.
  • Hi/Lo register split: 8 Lo regs (always available) + 8 Hi regs (incl. SP, LR, PC). Most ops only on Lo. ADD/BX/CMP/MOV have variants reaching Hi.
  • Branch addresses shifted by 1 (16-bit instructions). Branch-with-link is a 2-instruction sequence concatenating 11+11 bits, shifted left → 23-bit address.
  • Data-transfer offsets: 5 bits for GPRs, 8 bits for SP/PC.

12.14 Instructions Unique to SuperH

R0 plays a special role: indirect-indexed and PC-relative addressing add R0 to another register; only R0 can be the operand for immediate AND, CMP, OR, XOR.

  • DT — decrement and test (set T=1 if result is 0).
  • Optional delayed branch for BT/BF — unique among embedded RISCs (others don't use delayed branches).
  • Many multiply variants: MULS, MULU, DMULS, DMULU, MUL — product in MACL/MACH.
  • EXTU/EXTS — zero/sign extend byte or halfword inside a 32-bit register.
  • 1-bit constant shifts only — to fit into 16-bit instructions.
  • Dynamic shift (SHLD/SHAD) — variable shift; sign of amount selects direction.
  • Rotate — ROTL/ROTR set T to the rotated bit; ROTCL/ROTCR include T in the rotation.
  • SWAP — swap high/low bytes of a word, or two bytes of low halfword.
  • XTRCT — middle 32 bits of a register pair into another register.
  • Negate-with-carry.
  • PREF — cache prefetch.
  • TAS (test-and-set) — atomic lock primitive (load byte, set T from zero-test, set MSB of byte to 1, store back).

12.15 Instructions Unique to M32R

  • Slight VLIW: a bit in the first 16-bit instruction of a pair tells the hardware whether the pair can be executed in parallel.
  • All branch displacements shifted left 2 before adding to PC (low 2 PC bits cleared) → branches must target 32-bit-aligned instructions even though some instructions are 16 bits. Same restriction on branch-and-link return addresses.
  • VLIW pairing means the necessary nop padding doesn't usually cost extra cycles.
  • Mitsubishi claims overall code is only 7% larger than for the Motorola 6800.
  • Divide returns the remainder (not the quotient).

12.16 Instructions Unique to MIPS-16

Not a separate ISA — it's a 16-bit-instruction extension of full 32-bit MIPS, compatible with MIPS I/II/III/IV/V. ISA mode bit: 0 = 32-bit, 1 = 16-bit. JALX toggles the mode bit; JR/JALR set the mode bit from the branch register's MSB. Procedures must be all 16-bit or all 32-bit (except JAL/JALX, which are 32 bits even in 16-bit mode).

Key Differences from MIPS Core

  • Only 8 easy-to-use registers; explicit move instructions to access the other 24 of the underlying MIPS architecture.
  • SP treated as a separate register; many SP-relative data-transfer opcodes (LWSP, LDSP, SWSP, SDSP, ADJSP, DADJSP, ADDIUSPD, DADDIUSP).
  • Immediates shrunk to 5–8 bits.
  • EXTEND prefix — a 16-bit prefix (Intel-8086 style) prepended to extend an immediate or address field to 16 bits (does not need 32-bit alignment). Two exceptions: ADDIU/DADDIU have 4-bit fields; even with EXTEND, max is 15-bit.
  • EXTEND also extends 3-bit shift fields to 5 bits.
  • New addressing mode: PC-relative load (LWPC, LDPC) — 8-bit immediate shifted by 2/3, added to PC. Lets 32/64-bit constants be embedded in code. ADDIUPC computes a PC-relative address into a register.
  • 16-bit forms of 64-bit ops (data transfer, arithmetic, shifts) — MIPS-16 alone among embedded RISCs subsets a 64-bit address ISA.
  • Dropped: trapping-add variants (ADD, ADDI, SUB, DADD, DADDI, DSUB), logical immediates (ANDI, ORI, XORI), most compare-and-branch (BEQ, BNE, BGEZ, BGTZ, BLEZ, BLTZ) — replaced by T-register-based branches (BTEQZ, BTNEZ).
  • Branch addresses shifted by 1 (16-bit insts).
  • Delayed branches gone — branches take effect before next instruction. Jumps still have 1-slot delay.
  • Data-transfer offsets zero-extended in 16-bit mode (sign-extended only with EXTEND prefix). Shifted left 1/2/3 bits for halfword/word/doubleword.
  • Shift amount of 0 redefined to mean shift of 8 bits.
  • Added load immediate, negate, not (since r0 is no longer freely available).

12.17 Concluding Remarks

~90% of executed instructions on any of these 10 ISAs would map to the MIPS core subset. The 10 architectures share the most architectural agreement in the history of computing — contrast with the four 1970s architectures (which had little agreement on anything).

Lineage Highlights

  • CDC 6600 / Cray-1 — load-store ancestors with r0=0 and separate int/FP registers.
  • IBM 801 → unsuccessful RT/PC. America project → Power architecture.
  • Berkeley RISC → SPARC (Sun), inspiration for ARM.
  • Stanford MIPS → MIPS commercial.
  • HP PA-RISC inherited some 801 alumni.
  • DEC Alpha 21064 ≈ MIPS III + PRISM ideas. Alpha chip versions: EV4, EV45, EV5, EV56, EV6 (21064 → 21264). "EV" = "extended VAX."
  • All desktop RISCs evolved to 64-bit address architectures fairly painlessly.

Self-Test Q&A

Categories & high-level trends (5)
Name the 5 desktop/server RISCs and 5 embedded RISCs in the survey.
Desktop: Alpha, PA-RISC, PowerPC, MIPS-64, SPARC. Embedded: ARM, Thumb, SuperH, M32R, MIPS-16.
What's the fundamental register-count and instruction-width split between desktop and embedded RISCs?
Desktop: 32 GPRs, always 32-bit instructions. Embedded: 8–16 GPRs, 16- or 32-bit instructions.
Why are Thumb and MIPS-16 not really separate ISAs?
They're 16-bit modes of ARM and MIPS, switched into via a call/branch. Hardware translates Thumb back to ARM.
What rough percentage of executed instructions on these 10 ISAs would be MIPS-core-equivalent?
~90%.
Why is the MIPS jump/call constant not sign-extended?
It replaces only the lower 28 bits of PC, leaving the upper 4 bits unchanged.
Addressing modes & formats (4)
How is "register indirect" addressing synthesized on a desktop RISC?
As displacement addressing with offset = 0.
How is "absolute" addressing synthesized?
As displacement addressing with the always-zero register as the base.
Why are PC-relative branch offsets shifted left 2 bits on desktop RISCs?
Because all instructions are 32-bit aligned, the low two address bits are always 0 — using them in the offset would waste reach.
What is the ARM 8-bit immediate trick?
The 8-bit value is zero-extended to 32 then rotated right by an even count (2–30) chosen by 4 bits in the instruction. Captures all powers of two as single immediates.
Compare-and-branch strategies (8)
SPARC's mechanism?
Four PSW condition codes (N, Z, C, V) optionally set per instruction. Explicit compare = subtract with r0 destination. v9 added branches with prediction-hint bit.
PowerPC's mechanism?
Eight 4-bit condition-code registers. Compares can target any of them; CRAND/CROR/CRXOR combine them.
MIPS's mechanism?
No integer CCs. BEQ/BNE compare two GPRs; SLT/SLTI/SLTU/SLTIU set 1/0 in a register. Special compare-with-zero branches: BGEZ, BGTZ, BLEZ, BLTZ.
Alpha's mechanism?
CMPxx writes 1/0 into a GPR; FP compares write 2.0/0.0. Branches compare against 0 or test the LSB.
ARM's mechanism?
Four CCs optionally set. CMP, CMN (add), TST (AND), TEQ (XOR). Every instruction is conditional on the CCs.
SuperH's mechanism?
Single T-bit. BT branches if T=1, BF if T=0.
MIPS-16's mechanism?
SLT result goes into a special T register (= MIPS r24). BTEQZ/BTNEZ test T; BEQZ/BNEZ test other regs. Compare instructions added because BEQ/BNE were dropped.
Why does SPARC explicitly subtract into r0?
Because SPARC compares are arithmetic instructions that set CCs as a side effect; using r0 as destination throws away the result while still updating CCs.
Branch variants (5)
What is a delayed branch?
The instruction after the branch always executes, regardless of whether the branch is taken. Lets the pipeline finish the fetched instruction productively instead of squashing it.
Which RISCs use immediate (non-delayed) branches?
Alpha, PowerPC, and most embedded RISCs (SuperH offers it as an option).
What is SPARC annulling?
The delay-slot instruction executes only if the branch is taken — lets the compiler safely place the branch target in the slot.
What is MIPS "branch likely"?
A delayed branch that annuls the delay-slot instruction if the branch is NOT taken.
What is PA-RISC nullification?
Almost any instruction may annul the next instruction. Branches choose to nullify based on direction (forward/backward) and taken/not-taken — handy for loop bodies.
Multimedia (SIMD) extensions (5)
How does a single 64-bit ALU do parallel byte/halfword ops?
By partitioning (breaking) the carry chains so the ALU treats the 64 bits as eight 8-bit, four 16-bit, or two 32-bit lanes.
What is saturating arithmetic, and why does multimedia want it?
On overflow, clamp to MAX/MIN instead of wrapping (mod). For audio/video, clipping is far less destructive than wraparound.
Which extension does MIPS use, and what's its accumulator?
MDMX. 192-bit (3×64) accumulator partitioned to 8 bytes × 24 bits or 4 halfwords × 48 bits, used for add/sub/MAC.
Which desktop RISC was the multimedia holdout?
PowerPC.
Why are these instructions intended for hand-written libraries rather than compiler use?
Inconsistency across ISAs — a single multimedia compiler that produced good code for all of them would be very hard. Auto-vectorization is also brittle.
SPARC register windows (5)
What problem do register windows solve?
Reducing register save/restore traffic on procedure calls — each call gets a fresh window of registers.
How is unbounded call depth supported?
The windows form a circular buffer. When it overflows, hardware traps and the OS spills to the stack.
Typical window structure?
8 globals + 8 locals + 8 incoming + 8 outgoing parameters. 16 unique regs per window (overlap).
What do SAVE and RESTORE do besides change windows?
They are also add instructions — used to allocate/deallocate stack frames (SP arithmetic) without an extra cycle.
Why didn't register windows slow the SPARC clock as predicted?
Cache access dominates register access in real implementations, so the larger register file isn't on the critical path.
ARM specifics (5)
Most distinctive ARM feature?
Conditional execution of every instruction via a 4-bit condition field — turns instructions into NOPs when the condition fails.
Why does ARM use that rotated-8-bit-immediate scheme?
To pack a useful range of constants into 12 bits, including all powers of two — common values in embedded code.
What do LDM/STM do?
Block load/store: a 16-bit register mask in the instruction selects which of the 16 GPRs to load or store. Used for procedure entry/return and block memory copy.
Does ARM have integer divide?
No (in the v4 era covered here).
What does the BX instruction do?
Branch and exchange between ARM and Thumb modes. Lower 31 bits of register go to PC; the MSB selects ARM (1) or Thumb (0).
Alpha specifics (5)
Why was Alpha originally without byte/halfword loads/stores?
Architects believed byte loads needed an extra shifter and byte stores would force read-modify-write on ECC memory. Bytes/halfwords were added in Alpha 21164A.
What does TRAPB do?
Stalls until all prior arithmetic instructions are guaranteed to complete without exception — used to enforce precise FP exceptions. ~5× slowdown if used aggressively.
Why does Alpha need precise interrupts to handle NaN/inf/denormals?
Because those cases are completed in software, which must locate the offending instruction and operand values.
What is PAL code?
Privileged Architecture Library — privileged-mode code with interrupts off and VM mapping off. Used for TLB management, atomic ops, OS primitives. Called via CALL_PAL.
What's special about Alpha FP register F31?
Hardwired to 0 — simplifies compares to zero.
PowerPC, PA-RISC, others (8)
Why does PowerPC have a dedicated link register?
Return jumps don't need to read the GPR file → faster returns. Frees up a GPR.
What's PowerPC's count register for?
Loop counts. Branch-on-count auto-decrements and the value is visible to the branch unit early in the cycle (good prediction).
What does PA-RISC's nullification do?
Lets almost any instruction (arithmetic/logical/branch) skip the next instruction based on a condition — eliminates branches around single-instruction then bodies.
Why does PA-RISC have so many conditional branches?
Inherited from a CISC heritage; combined with nullification it offers many addressing/compare combinations directly. PA-RISC has the most addressing modes and instruction formats of any RISC.
SuperH "DT" instruction?
Decrement and test — decrements a register and sets T=1 if the result is 0. Common in loop epilogues.
M32R divide returns what?
The remainder, not the quotient.
M32R VLIW feature?
A bit in the first instruction of a 16-bit pair tells hardware whether the pair can execute in parallel.
What is MIPS-16's EXTEND prefix?
A 16-bit prefix prepended to a 16-bit MIPS-16 instruction to widen its immediate or address field to 16 bits (Intel-8086-inspired). Doesn't need 32-bit alignment.
Common extensions to MIPS core (4)
What does an "atomic swap" instruction do?
Exchanges a register and a memory location atomically — foundation for OS semaphores and multiprocessor sync.
What does a prefetch instruction do?
Provides an address and a hint (read/write soon, once vs many) so the cache can pull the line in early. Never traps.
What does the "Endian" big/little PSR bit do?
Lets the same chip act as big- or little-endian by complementing some low-order address bits in data transfers.
Why is "shared memory synchronization" (memory barrier) important?
In a cache-coherent multiprocessor it forces all loads/stores before the barrier to complete before any after — needed for sane ordering of synchronization sequences.

Common Exam Traps

  • "Thumb is a separate ISA from ARM." No. It's a 16-bit instruction mode of ARM, translated by hardware to ARM ops.
  • "MIPS-16 uses the same registers as MIPS-32." Partly. Hardware still has all 32 MIPS registers, but only 8 are easily addressable in 16-bit mode; explicit moves expose the rest.
  • "All RISCs reserve a register equal to zero." No. Desktop RISCs do; embedded RISCs typically do not. PowerPC's r0 is reserved as a base-only zero-source, not a true hardwired zero everywhere.
  • "All instructions in all RISCs are 32 bits." No. Desktop RISCs are 32-bit. Embedded RISCs use 16- or 32-bit; Thumb and MIPS-16 are pure 16-bit modes.
  • "PC-relative branch offsets are added to PC unshifted." No. Desktop RISCs shift by 2 (32-bit instructions); embedded by 1 (16-bit).
  • "SPARC condition codes are set on every arithmetic instruction." No. Setting CCs is optional per instruction (this is a pipelining-friendly choice).
  • "Every RISC has integer divide." No. Alpha and ARM (v4) lack integer divide.
  • "Multimedia extensions perform identically across desktop RISCs." No. The only universal ops are AND, OR, XOR. Different ISAs partition different lane widths and offer wildly different operations (Alpha's MAX is minimal; MIPS MDMX has a 192-bit accumulator).
  • "Saturating arithmetic produces a wraparound result on overflow." Backwards. Saturation clamps to MAX or MIN; modular (two's-complement) arithmetic wraps. Saturation is the DSP/multimedia behavior precisely because wraparound is destructive.
  • "PA-RISC's nullification only works on branches." No. It's generalized to almost any arithmetic/logical instruction or branch.
  • "ARM's conditional execution applies only to branches." No. Every ARM instruction has a 4-bit condition field. Conditional branches are just conditional unconditional-branches.
  • "Register windows make SPARC's clock slower than MIPS." In practice no. Cache access dominated and SPARC kept up with MIPS in clock for years.
  • "PowerPC r0 is hardwired to zero." No. r0 is treated as 0 only when used as the base in base+displacement; otherwise it's a normal register.
  • "Alpha's TRAPB is needed once per program." No. One TRAPB is needed per FP-exception-causing instruction to enforce precise exceptions, which is why the worst-case slowdown is ~5×.
  • "MIPS-16 supports mixing 16- and 32-bit instructions inside a single procedure." No. A procedure must be all one mode. Mode switches happen at procedure boundaries via JR/JALR/JALX.
